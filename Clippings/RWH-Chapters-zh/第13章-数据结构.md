# 第13章-数据结构

## 关联列表

通常，我们不得不处理无序但按键索引的数据。例如，Unix 管理员可能有一个数字 UID（用户 ID）和它们对应的文本用户名的列表。这个列表的价值在于能够根据给定 UID 查找文本用户名，而不是数据的顺序。换句话说，UID 是数据库的键。

在 Haskell 中，有几种方式处理以这种方式结构化的数据。两种最常见的是关联列表和 Data.Map 模块提供的 Map 类型。关联列表很方便，因为它们很简单。它们是标准 Haskell 列表，所以所有熟悉的列表函数都可以使用它们。然而，对于大型数据集，Map 将比关联列表具有相当大的性能优势。我们将在本章中使用两者。

关联列表只是包含（键，值）元组的普通列表。从 UID 到用户名的映射列表的类型可能是 [(Integer, String)]。我们几乎可以使用任何类型*作为键和值。

我们可以像构建任何其他列表一样构建关联列表。Haskell 带有一个名为 Data.List.lookup 的内置函数，用于在关联列表中查找数据。它的类型是 Eq a => a -> [(a, b)] -> Maybe b。你能从那个类型猜到它是如何工作的吗？让我们在 ghci 中看一下：

```ghci
> let al = [(1, "one"), (2, "two"), (3, "three"), (4, "four")]
> lookup 1 al
Just "one"
> lookup 5 al
Nothing
```

lookup 函数真的很简单。这里有一种我们可以编写它的方式：

```haskell
-- file: ch13/lookup.hs
myLookup :: Eq a => a -> [(a, b)] -> Maybe b
myLookup _ [] = Nothing
myLookup key ((thiskey,thisval):rest) =
    if key == thiskey
       then Just thisval
       else myLookup key rest
```

这个函数如果传入空列表返回 Nothing。否则，它将键与我们正在寻找的键进行比较。如果找到匹配，则返回对应的值；否则，搜索列表的其余部分。

让我们看一个更复杂的关联列表示例。在 Unix/Linux 机器上，有一个名为 /etc/passwd 的文件，存储用户名、UID、主目录和各种其他数据。我们将编写一个解析此类文件、创建关联列表并让用户通过 UID 查找用户名的程序：

```haskell
-- file: ch13/passwd-al.hs
import Data.List
import System.IO
import Control.Monad(when)
import System.Exit
import System.Environment(getArgs)
main = do
    -- Load the command-line arguments
    args <- getArgs
    -- If we don't have the right amount of args, give an error and abort
    when (length args /= 2) $ do
        putStrLn "Syntax: passwd-al filename uid"
        exitFailure
    -- Read the file lazily
    content <- readFile (args !! 0)
    -- Compute the username in pure code
    let username = findByUID content (read (args !! 1))
    -- Display the result
    case username of
         Just x -> putStrLn x
         Nothing -> putStrLn "Could not find that UID"
-- Given the entire input and a UID, see if we can find a username.
findByUID :: String -> Integer -> Maybe String
findByUID content uid =
    let al = map parseline . lines $ content
        in lookup uid al
-- Convert a colon-separated line into fields
parseline :: String -> (Integer, String)
parseline input =
    let fields = split ':' input
        in (read (fields !! 2), fields !! 0)
{- | Takes a delimiter and a list.  Break up the list based on the
-  delimiter. -}
split :: Eq a => a -> [a] -> [[a]]
-- If the input is empty, the result is a list of empty lists.
split _ [] = [[]]
split delim str =
    let -- Find the part of the list before delim and put it in "before".
        -- The rest of the list, including the leading delim, goes
        -- in "remainder".
        (before, remainder) = span (/= delim) str
        in
        before : case remainder of
                      [] -> []
                      x -> -- If there is more data to process,
                           -- call split recursively to process it
                           split delim (tail x)
```

让我们看看这个程序。它的核心是 findByUID，它是一个简单的函数，解析输入的每一行，然后对结果调用 lookup。剩余的程序关注解析输入。输入文件看起来像这样：

```
root:x:0:0:root:/root:/bin/bash
daemon:x:1:1:daemon:/usr/sbin:/bin/sh
bin:x:2:2:bin:/bin:/bin/sh
sys:x:3:3:sys:/dev:/bin/sh
sync:x:4:65534:sync:/bin:/bin/sync
games:x:5:60:games:/usr/games:/bin/sh
man:x:6:12:man:/var/cache/man:/bin/sh
lp:x:7:7:lp:/var/spool/lpd:/bin/sh
mail:x:8:8:mail:/var/mail:/bin/sh
news:x:9:9:news:/var/spool/news:/bin/sh
jgoerzen:x:1000:1000:John Goerzen,,,:/home/jgoerzen:/bin/bash
```

其字段用冒号分隔，包括用户名、数字用户 ID、数字组 ID、全名、主目录和 shell。没有字段可能包含内部冒号。

## Map

Data.Map 模块提供了一个 Map 类型，其行为类似于关联列表但性能更好。

Map 给了我们与其他语言中哈希表相同的能力。内部，map 实现为平衡二叉树。与哈希表相比，这对于具有不可变数据的语言是一种更有效的表示。这是纯函数式编程如何深刻影响我们编写代码方式的最明显例子：我们选择能够清晰表达且高效执行的数据结构和算法，但我们对特定任务的选择通常与命令式语言中的对应物不同。

Data.Map 模块中的一些函数与 Prelude 中的函数同名。因此，我们将使用 import qualified Data.Map as Map 导入它，并使用 Map.name 来引用该模块中的名称。让我们通过查看构建 map 的一些方法来开始我们对 Data.Map 的介绍：

```haskell
-- file: ch13/buildmap.hs
import qualified Data.Map as Map
-- Functions to generate a Map that represents an association list
-- as a map
al = [(1, "one"), (2, "two"), (3, "three"), (4, "four")]
{- | Create a map representation of 'al' by converting the association
-  list using Map.fromList -}
mapFromAL =
    Map.fromList al
{- | Create a map representation of 'al' by doing a fold -}
mapFold =
    foldl (\map (k, v) -> Map.insert k v map) Map.empty al
{- | Manually create a map with the elements of 'al' in it -}
mapManual =
    Map.insert 2 "two" .
    Map.insert 4 "four" .
    Map.insert 1 "one" .
    Map.insert 3 "three" $ Map.empty
```

像 Map.insert 这样的函数以通常的 Haskell 方式工作：它们返回输入数据的副本，并应用请求的更改。这对 map 非常方便。这意味着你可以使用 foldl 来构建 map，如 mapFold 示例。或者，你可以像 mapManual 示例中那样链接对 Map.insert 的调用。让我们用 ghci 验证所有这些 map 都符合预期：

```ghci
> :l buildmap.hs
[1 of 1] Compiling Main             ( buildmap.hs, interpreted )
Ok, modules loaded: Main.
> al
Loading package array-0.1.0.0 ... linking ... done.
Loading package containers-0.1.0.2 ... linking ... done.
[(1,"one"),(2,"two"),(3,"three"),(4,"four")]
> mapFromAL
fromList [(1,"one"),(2,"two"),(3,"three"),(4,"four")]
> mapFold
fromList [(1,"one"),(2,"two"),(3,"three"),(4,"four")]
> mapManual
fromList [(1,"one"),(2,"two"),(3,"three"),(4,"four")]
```

注意 mapManual 的输出与我们用来构造 map 的列表顺序不同。Map 不保证它们会保留原始顺序。

Map 在概念上与关联列表类似操作。Data.Map 模块提供了从 map 添加和删除数据的函数。它还允许我们过滤它们、修改它们、折叠它们以及与关联列表相互转换。该模块的库文档很好，所以而不是详细讨论每个函数，我们将呈现一个将本章中讨论的许多概念结合在一起的示例。

## 函数也是数据

Haskell 的力量之一是它让我们轻松创建和操作函数。让我们看一个将函数存储为其字段之一的记录：

```haskell
-- file: ch13/funcrecs.hs
{- | Our usual CustomColor type to play with -}
data CustomColor =
  CustomColor {red :: Int,
               green :: Int,
               blue :: Int}
  deriving (Eq, Show, Read)
{- | A new type that stores a name and a function.
The function takes an Int, applies some computation to it, and returns
an Int along with a CustomColor -}
data FuncRec =
    FuncRec {name :: String,
             colorCalc :: Int -> (CustomColor, Int)}
plus5func color x = (color, x + 5)
purple = CustomColor 255 0 255
plus5 = FuncRec {name = "plus5", colorCalc = plus5func purple}
always0 = FuncRec {name = "always0", colorCalc = \_ -> (purple, 0)}
```

注意 colorCalc 字段的类型：它是一个函数。它接受一个 Int 并返回 (CustomColor, Int) 的元组。我们创建两个 FuncRec 记录：plus5 和 always0。注意它们的 colorCalc 将始终返回紫色。FuncRec 本身没有字段存储颜色，但该值以某种方式成为函数本身的一部分。这称为闭包。让我们玩一下：

```ghci
> :l funcrecs.hs
[1 of 1] Compiling Main             ( funcrecs.hs, interpreted )
Ok, modules loaded: Main.
> :t plus5
plus5 :: FuncRec
> name plus5
"plus5"
> :t colorCalc plus5
colorCalc plus5 :: Int -> (CustomColor, Int)
> (colorCalc plus5) 7
(CustomColor {red = 255, green = 0, blue = 255},12)
> :t colorCalc always0
colorCalc always0 :: Int -> (CustomColor, Int)
> (colorCalc always0) 7
(CustomColor {red = 255, green = 0, blue = 255},0)
```

这工作得足够好，但你可能想知道如何做更高级的事情，例如在多个地方使数据可用。类型构造函数可能有用。这里有一个例子：

```haskell
-- file: ch13/funcrecs2.hs
data FuncRec =
    FuncRec {name :: String,
             calc :: Int -> Int,
             namedCalc :: Int -> (String, Int)}
mkFuncRec :: String -> (Int -> Int) -> FuncRec
mkFuncRec name calcfunc =
    FuncRec {name = name,
             calc = calcfunc,
             namedCalc = \x -> (name, calcfunc x)}
plus5 = mkFuncRec "plus5" (+ 5)
always0 = mkFuncRec "always0" (\_ -> 0)
```

这里我们有一个名为 mkFuncRec 的函数，它接受一个 String 和另一个函数作为参数，然后返回一个新的 FuncRec 记录。注意 mkFuncRec 的两个参数都在多个地方使用。让我们试试：

```ghci
> :l funcrecs2.hs
[1 of 1] Compiling Main             ( funcrecs2.hs, interpreted )
Ok, modules loaded: Main.
> :t plus5
plus5 :: FuncRec
> name plus5
"plus5"
> (calc plus5) 5
10
> (namedCalc plus5) 5
("plus5",10)
> let plus5a = plus5 {name = "PLUS5A"}
> name plus5a
"PLUS5A"
> (namedCalc plus5a) 5
("plus5",10)
```

注意 plus5a 的创建。我们更改了 name 字段，但没有更改 namedCalc 字段。这就是为什么 name 有新名称，但 namedCalc 仍然返回传递给 mkFuncRec 的名称；除非我们显式更改它，否则它不会更改。

## 扩展示例：/etc/passwd

为了说明多种不同数据结构一起使用的用法，我们准备了一个扩展示例。此示例解析并存储典型 /etc/passwd 文件格式的条目：

```haskell
-- file: ch13/passwdmap.hs
import Data.List
import qualified Data.Map as Map
import System.IO
import Text.Printf(printf)
import System.Environment(getArgs)
import System.Exit
import Control.Monad(when)
{- | The primary piece of data this program will store.
   It represents the fields in a POSIX /etc/passwd file -}
data PasswdEntry = PasswdEntry {
    userName :: String,
    password :: String,
    uid :: Integer,
    gid :: Integer,
    gecos :: String,
    homeDir :: String,
    shell :: String}
    deriving (Eq, Ord)
{- | Define how we get data to a 'PasswdEntry'. -}
instance Show PasswdEntry where
    show pe = printf "%s:%s:%d:%d:%s:%s:%s"
                (userName pe) (password pe) (uid pe) (gid pe)
                (gecos pe) (homeDir pe) (shell pe)
{- | Converting data back out of a 'PasswdEntry'. -}
instance Read PasswdEntry where
    readsPrec _ value =
        case split ':' value of
             [f1, f2, f3, f4, f5, f6, f7] ->
                 [(PasswdEntry f1 f2 (read f3) (read f4) f5 f6 f7, [])]
             x -> error $ "Invalid number of fields in input: " ++ show x
    where
    split :: Eq a => a -> [a] -> [[a]]
    split _ [] = [[]]
    split delim str =
        let (before, remainder) = span (/= delim) str
        in
        before : case remainder of
                      [] -> []
                      x -> split delim (tail x)
-- Convenience aliases; we'll have two maps: one from UID to entries
-- and the other from username to entries
type UIDMap = Map.Map Integer PasswdEntry
type UserMap = Map.Map String PasswdEntry
{- | Converts input data to maps.  Returns UID and User maps. -}
inputToMaps :: String -> (UIDMap, UserMap)
inputToMaps inp =
    (uidmap, usermap)
    where
    uidmap = Map.fromList . map (\pe -> (uid pe, pe)) $ entries
    usermap = Map.fromList .
              map (\pe -> (userName pe, pe)) $ entries
    entries = map read (lines inp)
main = do
    args <- getArgs
    when (length args /= 1) $ do
        putStrLn "Syntax: passwdmap filename"
        exitFailure
    content <- readFile (head args)
    let maps = inputToMaps content
    mainMenu maps
mainMenu maps@(uidmap, usermap) = do
    putStr optionText
    hFlush stdout
    sel <- getLine
    case sel of
         "1" -> lookupUserName >> mainMenu maps
         "2" -> lookupUID >> mainMenu maps
         "3" -> displayFile >> mainMenu maps
         "4" -> return ()
         _ -> putStrLn "Invalid selection" >> mainMenu maps
    where
    lookupUserName = do
        putStrLn "Username: "
        username <- getLine
        case Map.lookup username usermap of
             Nothing -> putStrLn "Not found."
             Just x -> print x
    lookupUID = do
        putStrLn "UID: "
        uidstring <- getLine
        case Map.lookup (read uidstring) uidmap of
             Nothing -> putStrLn "Not found."
             Just x -> print x
    displayFile =
        putStr . unlines . map (show . snd) . Map.toList $ uidmap
    optionText =
          "\npasswdmap options:\n\
          \\n\
          \1   Look up a user name\n\
          \2   Look up a UID\n\
          \3   Display entire file\n\
          \4   Quit\n\n\
          \Your selection: "
```

此示例维护两个 map：一个从用户名到 PasswdEntry，另一个从 UID 到 PasswdEntry。数据库开发者可能发现将其视为对数据有两个不同索引以加速对不同字段的搜索会很方便。

看看 PasswdEntry 的 Show 和 Read 实例。已经有一个将此类数据显示为字符串的标准格式：系统已经使用的冒号分隔版本。因此我们的 Show 函数以该格式显示 PasswdEntry，而 Read 解析该格式。

## 扩展示例：数值类型

我们已经告诉你 Haskell 的类型系统有多强大和表达力。我们已经向你展示了很多使用那种力量的方式。这里有一个真正看到这一点行动的机会。

回到第 144 页的"数值类型"，我们展示了 Haskell 附带的数值类型类。让我们看看通过定义新类型和利用数值类型类将它们与 Haskell 基础数学集成可以做什么。

首先让我们考虑一下在 ghci 中与我们新类型交互时我们希望看到什么。首先，也许将数值表达式呈现为字符串会很好，确保指示正确的优先级。也许我们可以创建一个名为 prettyShow 的函数来做到这一点。我们稍后会向你展示如何编写它，但首先我们会看看我们可能会如何使用它：

```ghci
> :l num.hs
[1 of 1] Compiling Main             ( num.hs, interpreted )
Ok, modules loaded: Main.
> 5 + 1 * 3
8
> prettyShow $ 5 + 1 * 3
"5+(1*3)"
> prettyShow $ 5 * 1 + 3
"(5*1)+3"
```

这看起来不错，但它不那么聪明。我们可以轻松简化表达式中的 1 * 部分。也许可以编写一个函数来进行一些非常基本的简化？

```ghci
> prettyShow $ simplify $ 5 + 1 * 3
"5+3"
```

关于将数值表达式转换为逆波兰记法（RPN）怎么样？RPN 是一种后缀 notation，不需要括号，通常出现在 HP 计算器上。RPN 是一种基于堆栈的 notation。我们将数字压入堆栈，当我们输入运算时，它们弹出最近的数字并将结果放入堆栈：

```ghci
> rpnShow $ 5 + 1 * 3
"5 1 3 * +"
> rpnShow $ simplify $ 5 + 1 * 3
"5 3 +"
```

也许能够用未知数的符号表示简单表达式会很好：

```ghci
> prettyShow $ 5 + (Symbol "x") * 3
"5+(x*3)"
```

在处理数字时跟踪单位通常很重要。例如，当你看到数字 5 时，它是指 5 米、5 英尺还是 5 字节？当然，如果你把 5 米除以 2 秒，系统应该能够弄清楚适当的单位。而且，它应该阻止你将 2 秒加到 5 米：

```ghci
> 5 / 2
2.5
> (units 5 "m") / (units 2 "s")
2.5_m/s
> (units 5 "m") + (units 2 "s")
*** Exception: Mis-matched units in add or subtract
> (units 5 "m") + (units 2 "m")
7_m
> (units 5 "m") / 2
2.5_m
> 10 * (units 5 "m") / (units 2 "s")
25.0_m/s
```

如果我们定义对所有数字都有效的表达式或函数，我们应该能够计算结果或呈现表达式。例如，如果我们定义 test 的类型为 Num a => a——并且，比如说，test = 2 * 5 + 3，那么我们应该能够做到这一点：

```ghci
> test
13
> rpnShow test
"2 5 * 3 +"
> prettyShow test
"(2*5)+3"
> test + 5
18
> prettyShow (test + 5)
"((2*5)+3)+5"
> rpnShow (test + 5)
"2 5 * 3 + 5 +"
```

既然我们有单位，我们应该能够处理一些基本三角学。许多这些操作在角度上操作。让我们确保我们能够处理度和弧度：

```ghci
> sin (pi / 2)
1.0
> sin (units (pi / 2) "rad")
1.0_1.0
> sin (units 90 "deg")
1.0_1.0
> (units 50 "m") * sin (units 90 "deg")
50.0_m
```

最后，我们应该能够将所有这些放在一起并组合不同类型的表达式：

```ghci
> ((units 50 "m") * sin (units 90 "deg")) :: Units (SymbolicManip Double)
50.0*sin(((2.0*pi)*90.0)/360.0)_m
> prettyShow $ dropUnits $ (units 50 "m") * sin (units 90 "deg")
"50.0*sin(((2.0*pi)*90.0)/360.0)"
> rpnShow $ dropUnits $ (units 50 "m") * sin (units 90 "deg")
"50.0 2.0 pi * 90.0 * 360.0 / sin *"
> (units (Symbol "x") "m") * sin (units 90 "deg")
x*sin(((2.0*pi)*90.0)/360.0)_m
```

你刚刚看到的一切都可以用 Haskell 类型和类来实现。事实上，你一直在阅读一个真正的 ghci 会话，演示 num.hs，我们稍后会看到。

### 第一步

让我们思考我们将如何完成刚才展示的所有事情。首先，我们可以使用 ghci 检查 (+) 的类型，它是 Num a => a -> a -> a。如果我们想使加法运算符有一些自定义行为成为可能，那么我们将必须定义一个新类型并使其成为 Num 的实例。这个类型需要以符号方式存储表达式。我们可以从考虑加法等操作开始。为了存储它，我们需要存储操作本身及其左右两侧。左右两侧本身也可以是表达式。

因此，我们可以将表达式视为一种树。让我们从一些简单的类型开始：

```haskell
-- file: ch13/numsimple.hs
-- The "operators" that we're going to support
data Op = Plus | Minus | Mul | Div | Pow
        deriving (Eq, Show)
{- The core symbolic manipulation type -}
data SymbolicManip a =
          Number a           -- Simple number, such as 5
        | Arith Op (SymbolicManip a) (SymbolicManip a)
          deriving (Eq, Show)
{- SymbolicManip will be an instance of Num.  Define how the Num
operations are handled over a SymbolicManip.  This will implement things
like (+) for SymbolicManip. -}
instance Num a => Num (SymbolicManip a) where
    a + b = Arith Plus a b
    a - b = Arith Minus a b
    a * b = Arith Mul a b
    negate a = Arith Mul (Number (-1)) a
    abs a = error "abs is unimplemented"
    signum _ = error "signum is unimplemented"
    fromInteger i = Number (fromInteger i)
```

首先，我们定义一个名为 Op 的类型，它简单地表示我们将支持的一些操作。接下来，为 SymbolicManip a 有一个定义。由于 Num a 约束，任何 Num 都可以用于 a。因此完整类型可能是像 SymbolicManip Int 这样的东西。

SymbolicManip 类型可以是普通数字或一些算术运算。Arith 构造函数的类型是递归的，这在 Haskell 中是完全合法的。Arith 从一个 Op 和另外两个 SymbolicManip 项目创建一个 SymbolicManip。让我们看一个例子：

```ghci
> :l numsimple.hs
[1 of 1] Compiling Main             ( numsimple.hs, interpreted )
Ok, modules loaded: Main.
> Number 5
Number 5
> :t Number 5
Number 5 :: (Num t) => SymbolicManip t
> :t Number (5::Int)
Number (5::Int) :: SymbolicManip Int
> Number 5 * Number 10
Arith Mul (Number 5) (Number 10)
> (5 * 10)::SymbolicManip Int
Arith Mul (Number 5) (Number 10)
> (5 * 10 + 2)::SymbolicManip Int
Arith Plus (Arith Mul (Number 5) (Number 10)) (Number 2)
```

你可以看到我们已经有了一个非常基本的表达式表示。注意到 Haskell 如何"转换"5 * 10 + 2 为 SymbolicManip，甚至正确处理了求值顺序。这不是一个真正的转换；SymbolicManip 现在是一类数字。整数数值字面量在内部无论如何都被视为包装在 fromInteger 中，所以 5 作为 SymbolicManip Int 和作为 Int 一样有效。

从这里开始，我们的任务很简单：扩展 SymbolicManip 类型以表示我们将执行的所有操作，为其他数值类型类实现实例，并为我们自己的 Show 实例实现 SymbolicManip，以更易访问的方式呈现此树。

### 完成代码

这是完整的 num.hs，用于本章开头的 ghci 示例。让我们一次看一段这段代码：

```haskell
-- file: ch13/num.hs
import Data.List
--------------------------------------------------
-- Symbolic/units manipulation
--------------------------------------------------
-- The "operators" that we're going to support
data Op = Plus | Minus | Mul | Div | Pow
        deriving (Eq, Show)
{- The core symbolic manipulation type.  It can be a simple number,
a symbol, a binary arithmetic operation (such as +), or a unary
arithmetic operation (such as cos)
Notice the types of BinaryArith and UnaryArith: it's a recursive
type.  So, we could represent a (+) over two SymbolicManips. -}
data SymbolicManip a =
          Number a           -- Simple number, such as 5
        | Symbol String      -- A symbol, such as x
        | BinaryArith Op (SymbolicManip a) (SymbolicManip a)
        | UnaryArith String (SymbolicManip a)
          deriving (Eq)
```

在这部分代码中，我们定义了一个与之前使用的相同的 Op。我们还定义了 SymbolicManip，它与之前使用的类似。在这个版本中，我们现在支持一元算术运算（如 abs 或 cos 等仅需要一个参数的操作）。接下来我们为 Num 定义实例：

```haskell
-- file: ch13/num.hs
{- SymbolicManip will be an instance of Num.  Define how the Num
operations are handled over a SymbolicManip.  This will implement things
like (+) for SymbolicManip. -}
instance Num a => Num (SymbolicManip a) where
    a + b = BinaryArith Plus a b
    a - b = BinaryArith Minus a b
    a * b = BinaryArith Mul a b
    negate a = BinaryArith Mul (Number (-1)) a
    abs a = UnaryArith "abs" a
    signum _ = error "signum is unimplemented"
    fromInteger i = Number (fromInteger i)
```

这相当直接，也类似于我们之前的代码。注意之前我们无法正确支持 abs，但现在有 UnaryArith 构造函数就可以了。接下来我们定义更多实例：

这部分代码定义了一些相当直接的 Fractional 和 Floating 实例。现在让我们处理将表达式转换为字符串以进行显示：

```haskell
-- file: ch13/num.hs
{- Show a SymbolicManip as a String, using conventional
algebraic notation -}
prettyShow :: (Show a, Num a) => SymbolicManip a -> String
-- Show a number or symbol as a bare number or serial
prettyShow (Number x) = show x
prettyShow (Symbol x) = x
prettyShow (BinaryArith op a b) =
    let pa = simpleParen a
        pb = simpleParen b
        pop = op2str op
        in pa ++ pop ++ pb
prettyShow (UnaryArith opstr a) =
    opstr ++ "(" ++ show a ++ ")"
op2str :: Op -> String
op2str Plus = "+"
op2str Minus = "-"
op2str Mul = "*"
op2str Div = "/"
op2str Pow = "**"
{- Add parentheses where needed.  This function is fairly conservative
and will add parenthesis when not needed in some cases.

Haskell will have already figured out precedence for us while building
up the SymbolicManip. -}
simpleParen :: (Show a, Num a) => SymbolicManip a -> String
simpleParen (Number x) = prettyShow (Number x)
simpleParen (Symbol x) = prettyShow (Symbol x)
simpleParen x@(BinaryArith _ _ _) = "(" ++ prettyShow x ++ ")"
simpleParen x@(UnaryArith _ _) = prettyShow x
{- Showing a SymbolicManip calls the prettyShow function on it -}
instance (Show a, Num a) => Show (SymbolicManip a) where
    show a = prettyShow a
```

我们首先定义一个函数 prettyShow。它使用传统样式呈现表达式。算法相当简单：裸数字和符号呈现为裸形式；二元算术运算呈现为两侧加中间运算符，当然我们也处理一元运算符。op2str 简单地将 Op 转换为 String。在 simpleParen 中，我们有一个非常保守的算法，可以保守地添加括号以保持结果中的优先级清楚。最后，我们使用 prettyShow 使 SymbolicManip 成为 Show 的实例。现在让我们实现一个将表达式转换为 RPN 格式字符串的算法：

```haskell
-- file: ch13/num.hs
{- Show a SymbolicManip using RPN.  HP calculator users may
find this familiar. -}
rpnShow :: (Show a, Num a) => SymbolicManip a -> String
rpnShow i =
    let toList (Number x) = [show x]
        toList (Symbol x) = [x]
        toList (BinaryArith op a b) = toList a ++ toList b ++
           [op2str op]
        toList (UnaryArith op a) = toList a ++ [op]
        join :: [a] -> [[a]] -> [a]
        join delim l = concat (intersperse delim l)
    in join " " (toList i)
```

RPN 的粉丝会注意到，与使用传统 notation 呈现的算法相比，这个算法简单得多。特别是，我们不必担心在哪里添加括号，因为 RPN 根据定义只能以一种方式求值。接下来，让我们看看如何实现一个对表达式进行一些基本简化的函数：

```haskell
-- file: ch13/num.hs
{- Perform some basic algebraic simplifications on a SymbolicManip. -}
simplify :: (Num a) => SymbolicManip a -> SymbolicManip a
simplify (BinaryArith op ia ib) =
    let sa = simplify ia
        sb = simplify ib
        in
        case (op, sa, sb) of
                (Mul, Number 1, b) -> b
                (Mul, a, Number 1) -> a
                (Mul, Number 0, b) -> Number 0
                (Mul, a, Number 0) -> Number 0
                (Div, a, Number 1) -> a
                (Plus, a, Number 0) -> a
                (Plus, Number 0, b) -> b
                (Minus, a, Number 0) -> a
                _ -> BinaryArith op sa sb
simplify (UnaryArith op a) = UnaryArith op (simplify a)
simplify x = x
```

这个函数相当简单。对于某些二元算术运算——例如，将任何值乘以 1——我们能够轻松简化情况。首先，我们获得计算两侧的简化版本（这是递归发生的地方），然后简化结果。对于一元运算符，我们几乎不需要做什么，所以我们只简化它们所操作的表达式。

从这里开始，我们将为已建立的库添加单位支持。这将让我们表示诸如"5 米"之类的量。我们像之前一样从定义一个类型开始：

```haskell
-- file: ch13/num.hs
{- New data type: Units.  A Units type contains a number
and a SymbolicManip, which represents the units of measure.
A simple label would be something like (Symbol "m") -}
data Num a => Units a = Units a (SymbolicManip a)
           deriving (Eq)
```

所以，Units 包含一个数字和一个本身是 SymbolicManip 的标签。接下来，看到 Units 的 Num 实例可能不会令人惊讶：

```haskell
-- file: ch13/num.hs
{- Implement Units for Num.  We don't know how to convert between
arbitrary units, so we generate an error if we try to add numbers with
different units.  For multiplication, generate the appropriate
new units. -}
instance (Num a) => Num (Units a) where
    (Units xa ua) + (Units xb ub)
        | ua == ub = Units (xa + xb) ua
        | otherwise = error "Mis-matched units in add or subtract"
    (Units xa ua) - (Units xb ub) = (Units xa ua) + (Units (xb * (-1)) ub)
    (Units xa ua) * (Units xb ub) = Units (xa * xb) (ua * ub)
    negate (Units xa ua) = Units (negate xa) ua
    abs (Units xa ua) = Units (abs xa) ua
    signum (Units xa _) = Units (signum xa) (Number 1)
    fromInteger i = Units (fromInteger i) (Number 1)
```

现在可能清楚为什么我们使用 SymbolicManip 而不是 String 来存储测量单位。因为随着乘法等计算的发生，测量单位也会发生变化。例如，如果我们把 5 米乘以 2 米，我们得到 10 平方米。

我们强制加法的单位匹配，并以加法实现减法。让我们看更多 Units 的类型类实例：

-- file: ch13/num.hs
{- A simple function that takes a number and a String and returns an
appropriate Units type to represent the number and its unit of measure -}
units :: (Num z) => z -> String -> Units z
units a b = Units a (Symbol b)
{- Extract the number only out of a Units type -}
dropUnits :: (Num z) => Units z -> z
dropUnits (Units x _) = x
