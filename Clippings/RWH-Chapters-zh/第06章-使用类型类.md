# 第06章-使用类型类

类型类是 Haskell 最强大的特性之一。它们允许我们定义通用接口，提供各种类型的常见功能集。类型类是一些基本语言特性的核心，如相等性测试和数值运算符。但是在我们谈论类型类究竟是什么之前，我们想解释一下对它们的需求。

## 类型类的需求

假设由于某种难以理解的原因，Haskell 语言的设计者忘记实现相等测试 == 。一旦你从震惊中恢复，你就会着手实现自己的相等测试。你的应用程序由一个简单的 Color 类型组成，所以你第一个相等测试是这个类型。你的第一次尝试可能看起来像这样：

```haskell
-- file: ch06/naiveeq.hs
data Color = Red | Green | Blue

colorEq :: Color -> Color -> Bool
colorEq Red   Red   = True
colorEq Green Green = True
colorEq Blue  Blue  = True
colorEq _     _     = False
```

你可以用 ghci 测试：

```ghci
ghci> :load naiveeq.hs
[1 of 1] Compiling Main             ( naiveeq.hs, interpreted )
Ok, modules loaded: Main.
ghci> colorEq Red Red
True
ghci> colorEq Red Green
False
```

现在，假设你想为字符串添加相等测试。因为 Haskell String 是字符列表，我们可以写一个简单的函数来执行该测试。为简单起见，我们作弊一点，这里使用 == 运算符来说明：

```haskell
-- file: ch06/naiveeq.hs
stringEq :: [Char] -> [Char] -> Bool
-- Match if both are empty
stringEq [] [] = True
-- If both start with the same char, check the rest
stringEq (x:xs) (y:ys) = x == y && stringEq xs ys
-- Everything else doesn't match
stringEq _ _ = False
```

你现在应该能看到一个问题：我们必须为想要能够比较的每种不同类型使用不同名称的函数。这是低效和烦人的。使用 == 来比较任何东西要方便得多。编写通用函数如 /= 也可能有用，它可以在 == 方面实现，适用于几乎任何东西。通过拥有可以比较任何东西的通用函数，我们也可以使我们的代码通用：如果一段代码只需要比较东西，那么它应该能够接受编译器知道如何比较的任何数据类型。更重要的是，如果以后添加新数据类型，现有代码不必修改。

Haskell 的类型类旨在解决所有这些问题。

## 什么是类型类？

类型类定义了一组函数，可以根据给定的数据类型有不同的实现。类型类可能看起来像面向对象编程的对象，但它们确实相当不同。

让我们使用类型类解决上一节的相等困境。首先，我们必须定义类型类本身。我们需要一个接受两个参数的函数，两者类型相同，返回一个 Bool 表示它们是否相等。我们不关心那个类型是什么，但我们只需要该类型的两个项。这是我们第一个类型类定义：

```haskell
-- file: ch06/eqclasses.hs
class BasicEq a where
    isEqual :: a -> a -> Bool
```

这表示我们正在声明一个名为 BasicEq 的类型类，我们将用字母 a 来指代实例类型。类型的实例是实现类型类中定义的函数的任何类型。此类型类定义了一个函数。该函数接受两个参数——两者都对应于实例类型——并返回一个 Bool。

### 什么时候类不是类？

定义类型类的 Haskell 关键字是 class。不幸的是，这对于来自面向对象背景的你们来说可能会引起困惑，因为我们真的不是在定义相同的东西。

在第一行，参数 a 的名称是任意选择的——我们可以使用任何名称。关键是，当您列出函数的类型时，必须使用该名称来指代实例类型。

让我们在 ghci 中查看这个。回想一下，你可以在 ghci 中键入 :type 让它显示某物的类型。让我们看看它对 isEqual 说什么：

```ghci
*Main> :type isEqual
isEqual :: (BasicEq a) => a -> a -> Bool
```

你可以这样理解："对于所有类型 a，只要 a 是 BasicEq 的实例，isEqual 接受两个类型 a 的参数并返回一个 Bool。"让我们快速看一下为特定类型定义 isEqual：

```haskell
-- file: ch06/eqclasses.hs
instance BasicEq Bool where
    isEqual True  True  = True
    isEqual False False = True
    isEqual _     _     = False
```

你也可以用 ghci 验证我们现在可以对 Bool 使用 isEqual，但不能对任何其他类型：

```ghci
ghci> :load eqclasses.hs
[1 of 1] Compiling Main             ( eqclasses.hs, interpreted )
Ok, modules loaded: Main.
ghci> isEqual False False
True
ghci> isEqual False True
False
ghci> isEqual "Hi" "Hi"
<interactive>:1:0:
    No instance for (BasicEq [Char])
      arising from a use of `isEqual' at <interactive>:1:0-16
    Possible fix: add an instance declaration for (BasicEq [Char])
    In the expression: isEqual "Hi" "Hi"
    In the definition of `it': it = isEqual "Hi" "Hi"
```

注意，当我们尝试比较两个字符串时，ghci 认识到我们没有为 String 提供 BasicEq 的实例。因此它不知道如何比较 String，并建议我们可以通过为 [Char]（与 String 相同）定义 BasicEq 的实例来修复问题。

### 不等式函数

在这个例子中，不等式函数可能有用。以下是我们可以定义具有两个函数的类型类的方法：

```haskell
-- file: ch06/eqclasses.hs
class BasicEq2 a where
    isEqual2    :: a -> a -> Bool
    isNotEqual2 :: a -> a -> Bool
```

提供 BasicEq2 实例的人将需要定义两个函数：isEqual2 和 isNotEqual2。

虽然我们对 BasicEq2 的定义很好，但似乎我们为自己做了额外的工作。从逻辑上讲，如果我们知道 isEqual 或 isNotEqual 将返回什么，我们就知道如何找出其他函数对所有类型返回什么。不让类型类的用户为所有类型定义两个函数，我们可以为它们提供默认实现。这样，用户只需要实现一个函数。* 以下是显示如何执行此操作的示例：

```haskell
-- file: ch06/eqclasses.hs
class BasicEq3 a where
    isEqual3 :: a -> a -> Bool
    isEqual3 x y = not (isNotEqual3 x y)
    isNotEqual3 :: a -> a -> Bool
    isNotEqual3 x y = not (isEqual3 x y)
```

实现此类的用户必须提供至少一个函数的实现。如果他们愿意，可以实现两个，但不会被要求。虽然我们为两个函数都提供了默认实现，但每个函数依赖于另一个的存在来计算答案。如果我们没有指定至少一个，生成的代码将是一个无尽的循环。因此，至少一个函数必须始终被实现。

使用 BasicEq3，我们提供了一个与 Haskell 内置 == 和 /= 运算符功能非常相同的类。实际上，这些运算符是由一个看起来与 BasicEq3 几乎相同的类型类定义的。Haskell 98 报告定义了一个实现相等比较的类型类。以下是内置 Eq 类型类的代码。注意它与我们的 BasicEq3 类型类多么相似：

```haskell
class  Eq a  where
    (==), (/=) :: a -> a -> Bool
       -- Minimal complete definition:
       --     (==) or (/=)
    x /= y     =  not (x == y)
    x == y     =  not (x /= y)
```

## 声明类型类实例

现在你知道如何定义类型类，是时候学习如何定义类型类的实例了。回想一下，类型是通过实现该类型类所需的函数成为特定类型类的实例的。

回想一下我们在第 135 页的"类型类的需求"中创建对 Color 类型进行相等测试的尝试。现在让我们看看如何使相同的 Color 类型成为 BasicEq3 类的成员：

```haskell
-- file: ch06/eqclasses.hs
instance BasicEq3 Color where
    isEqual3 Red Red = True
    isEqual3 Green Green = True
    isEqual3 Blue Blue = True
    isEqual3 _ _ = False
```

注意，我们提供的函数基本上与第 135 页的"类型类的需求"中使用的相同。实际上，实现是相同的。但是，在这种情况下，我们可以在任何声明为 BasicEq3 实例的类型上使用 isEqual3，而不仅仅是这一个颜色类型。我们可以使用相同的基本模式为从数字到图形的任何内容定义相等测试。实际上，正如你将在第 148 页的"相等、排序和比较"中看到的，这正是你可以使 Haskell 的 == 运算符适用于你自己的自定义类型的方式。

还要注意 BasicEq3 类定义了 isEqual3 和 isNotEqual3，但我们在 Color 实例中只实现了其中一个。这是因为 BasicEq3 中包含的默认实现。因为我们没有显式定义 isNotEqual3，编译器自动使用 BasicEq3 声明中给出的默认实现。

## 重要的内置类型类

现在我们已经讨论了定义你自己的类型类和使你的类型成为类型类的实例，是时候向你介绍作为 Haskell Prelude 标准部分的类型类了。正如我们在本章开头提到的，类型类是语言一些重要方面的核心。我们将在此介绍最常见的类型类。更详细的资料，Haskell 库参考是一个很好的资源。它会给你类型类的描述，通常也会告诉你必须实现哪些函数才能有完整的定义。

### Show

Show 类型类用于将值转换为字符串。它最常用于将数字转换为字符串，但它为许多类型定义，所以它可以用于更多。如果你定义了自己的类型，使它们成为 Show 的实例将使在 ghci 中显示它们或在程序中打印它们变得容易。

Show 最重要的函数是 show。它接受一个参数——要转换的数据。它返回表示该数据的 String。ghci 这样报告 show 的类型：

```ghci
ghci> :type show
show :: (Show a) => a -> String
```

让我们看一些将值转换为字符串的示例：

```ghci
ghci> show 1
"1"
ghci> show [1, 2, 3]
"[1,2,3]"
ghci> show (1, 2)
"(1,2)"
```

记住 ghci 显示结果就像它们会被输入 Haskell 程序一样。所以表达式 show 1 返回包含数字 1 的单字符字符串。也就是说，引号不是字符串本身的一部分。我们可以通过使用 putStrLn 来明确这一点：

```ghci
ghci> putStrLn (show 1)
1
ghci> putStrLn (show [1,2,3])
[1,2,3]
```

你也可以对字符串使用 show：

```ghci
ghci> show "Hello!"
"\"Hello!\""
ghci> putStrLn (show "Hello!")
"Hello!"
ghci> show ['H', 'i']
"\"Hi\""
ghci> putStrLn (show "Hi")
"Hi"
ghci> show "Hi, \"Jane\""
"\"Hi, \\\"Jane\\\"\""
ghci> putStrLn (show "Hi, \"Jane\"")
"Hi, \"Jane\""
```

对字符串运行 show 可能会令人困惑。因为 show 生成适合机器用 Read 解析回的 Haskell 字面量的结果，它添加了适合包含在 Haskell 程序中的引号和转义。ghci 也使用 show 显示结果，所以引号和转义会被添加两次。使用 putStrLn 可以帮助明确这种区别。

你可以轻松为你的类型定义 Show 实例。这是一个示例：

```haskell
-- file: ch06/eqclasses.hs
instance Show Color where
    show Red   = "Red"
    show Green = "Green"
    show Blue  = "Blue"
```

此示例为我们的类型 Color（参见第 135 页的"类型类的需求"）定义 Show 的实例。实现很简单：我们定义一个 show 函数。这就是需要的全部。

### Show 类型类

Show 通常用于为数据定义字符串表示，适合机器用 Read 解析回来。如果此表示与通过 Show 的预期不同，Haskell 程序员通常会编写自定义函数来为最终用户格式化数据。

### Read

Read 类型类本质上是 Show 的反面。它定义函数，将获取字符串，解析它，并以是 Read 成员的任意类型返回数据。Read 中最有用的函数是 read。你可以像这样询问 ghci 它的类型：

```ghci
ghci> :type read
read :: (Read a) => String -> a
```

这是一个说明使用 read 和 show 的简单示例：

```haskell
-- file: ch06/read.hs
main = do
        putStrLn "Please enter a Double:"
        inpStr <- getLine
        let inpDouble = (read inpStr)::Double
        putStrLn ("Twice " ++ show inpDouble ++ " is " ++ show (inpDouble * 2))
```

这是 read 和 show 在一起的简单示例。请注意，在处理 read 时，我们给了 Double 的显式类型。这是因为 read 返回类型 Read a => a 的值，而 show 期望类型 Show a => a 的值。有许多类型为 Read 和 Show 都定义了实例。在不知道具体类型的情况下，编译器必须从这些许多类型中猜测需要哪一个。在这样的情况下，它通常可能选择 Integer。如果我们想接受浮点输入，这不起作用，所以我们提供显式类型。

### 关于默认化

在大多数情况下，如果省略显式 Double 类型注解，编译器会拒绝猜测一个常见类型并简单地给出错误。它能够在此默认为 Integer 是特殊情况，源于字面量 2 被视为 Integer，除非对它的类型有不同预期。

### Read 实例

我们可以为自定义类型定义 Read 实例。假设我们有这个 Color 类型：

```haskell
data Color = Red | Green | Blue
```

我们可以这样为它定义 Read 实例：

```haskell
instance Read Color where
    readPrec = do
        tok <- get
        case tok of
            "Red"   -> return Red
            "Green" -> return Green
            "Blue"  -> return Blue
            _       -> pfail
```

## 相等、排序和比较

Eq 类型类提供相等和不相等测试：

```haskell
class Eq a where
    (==), (/=) :: a -> a -> Bool
    x /= y = not (x == y)
    x == y = not (x /= y)
```

大多数 Prelude 类型是 Eq 的实例。

### Ord

Ord 类型类用于可以排序的值：

```ghci
ghci> :info Ord
class Eq a => Ord a where
    compare :: a -> a -> Ordering
    (<), (<=), (>=), (>) :: a -> a -> Bool
    max, min :: a -> a -> a
```

Ordering 类型是带有三个值的数据类型：LT、EQ 和 GT，分别表示小于、等于和大于。

### Num

Num 类型类是数字类型：

```ghci
ghci> :info Num
class (Eq a, Show a) => Num a where
    (+), (-), (*) :: a -> a -> a
    negate :: a -> a
    abs, signum :: a -> a
    fromInteger :: Integer -> a
```

### 派生派生

Haskell 允许我们使用 deriving 关键字自动为某些类型类创建实例：

```haskell
data Color = Red | Green | Blue
    deriving (Eq, Ord, Show, Read)
```

这会自动创建 Eq、Ord、Show 和 Read 的实例。

## 练习

1. 定义一个类型类 SimpleEQ，具有函数 isEqual，类型为 a -> a -> Bool。为 Color 类型创建此类型类的实例。

2. 修改 SimpleEQ 以提供 isNotEqual 的默认实现。

3. 为 Color 类型派生 Eq 实例，而不是手动定义。

4. 解释为什么 Eq 的实例不能自动派生用于包含函数的类型。

## 总结

类型类是 Haskell 强大的特性，允许我们：

- 定义通用接口
- 为不同类型提供相同函数的不同实现
- 使用派生自动创建实例
- 利用默认实现减少样板代码

理解类型类对于有效使用 Haskell 至关重要。
