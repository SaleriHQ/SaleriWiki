## 第二十七章：套接字与系统日志

## 基本网络编程

在本书的前几章中，我们讨论了通过网络运行的服务。两个例子是客户端/服务器数据库和 Web 服务。当需要设计新协议或与 Haskell 中没有现有辅助库的协议通信时，你需要使用 Haskell 库中的低级网络工具。

在本章中，我们将讨论这些低级工具。网络通信是一个广泛的话题，有整本书专门介绍它。我们将向你展示如何使用 Haskell 来应用你已经拥有的低级网络知识。

Haskell 的网络函数几乎总是直接对应于熟悉的 C 函数调用。由于大多数其他语言也建立在 C 之上，你会发现这个接口很熟悉。

## 使用 UDP 通信

UDP 将数据分解为数据包。它不保证数据到达目的地或只到达一次。它使用校验和来确保到达的数据包没有被损坏。UDP 倾向于用于对性能或延迟敏感的应用，其中每个单独的数据包比系统的整体性能更重要。它也可用于 TCP 行为不是最有效的情况，例如发送短的非连续消息。倾向于使用 UDP 的系统示例包括音频和视频会议、时间同步、基于网络的文件系统和日志系统。

## UDP 客户端示例：syslog

传统的 Unix syslog 服务允许程序通过网络将日志消息发送到记录它们的中央服务器。有些程序对性能相当敏感，可能会产生大量消息。在这些程序中，让日志记录施加最小的性能开销比保证每条消息都被记录更重要。此外，即使日志服务器无法访问，也希望程序继续运行。因此，UDP 是 syslog 支持用于传输日志消息的协议之一。协议很简单；我们在这里提供一个 Haskell 客户端实现：

```haskell
-- file: ch27/syslogclient.hs
import Data.Bits
import Network.Socket
import Network.BSD
import Data.List
import SyslogTypes
data SyslogHandle =
    SyslogHandle {slSocket :: Socket,
                  slProgram :: String,
                  slAddress :: SockAddr}
openlog :: HostName             -- ^ Remote hostname, or localhost
        -> String               -- ^ Port number or name; 514 is default
        -> String               -- ^ Name to log under
        -> IO SyslogHandle      -- ^ Handle to use for logging
openlog hostname port progname =
    do -- Look up the hostname and port.  Either raises an exception
       -- or returns a nonempty list.  First element in that list
       -- is supposed to be the best option.
       addrinfos <- getAddrInfo Nothing (Just hostname) (Just port)
       let serveraddr = head addrinfos
       -- Establish a socket for communication
       sock <- socket (addrFamily serveraddr) Datagram defaultProtocol
       -- Save off the socket, program name, and server address in a handle
       return $ SyslogHandle sock progname (addrAddress serveraddr)
syslog :: SyslogHandle -> Facility -> Priority -> String -> IO ()
syslog syslogh fac pri msg =
    sendstr sendmsg
    where code = makeCode fac pri
          sendmsg = "<" ++ show code ++ ">" ++ (slProgram syslogh) ++
                    ": " ++ msg
          -- Send until everything is done
          sendstr :: String -> IO ()
          sendstr [] = return ()
          sendstr omsg = do sent <- sendTo (slSocket syslogh) omsg
                                    (slAddress syslogh)
                            sendstr (genericDrop sent omsg)
closelog :: SyslogHandle -> IO ()
closelog syslogh = sClose (slSocket syslogh)
{- | Convert a facility and a priority into a syslog code -}
makeCode :: Facility -> Priority -> Int
makeCode fac pri =
    let faccode = codeOfFac fac
        pricode = fromEnum pri
        in
          (faccode `shiftL` 3) .|. pricode
```

module SyslogTypes where

            DEBUG                   -- ^ Debug messages
          | INFO                    -- ^ Information
          | NOTICE                  -- ^ Normal runtime conditions
          | WARNING                 -- ^ General Warnings
          | ERROR                   -- ^ General Errors
          | CRITICAL                -- ^ Severe situations
          | ALERT                   -- ^ Take immediate action

                    deriving (Eq, Ord, Show, Read, Enum)

              | LOCAL5
              | LOCAL6

                deriving (Eq, Show, Read)

                       (SYSLOG, 5),

                       (NEWS, 7),

codeOfFac :: Facility -> Int

                Just x -> x

                Just x -> x

## UDP Syslog 服务器

UDP 服务器将绑定到服务器计算机上的特定端口。它们将接受定向到该端口的数据包并处理它们。由于 UDP 是无状态的、面向数据包的协议，程序员通常使用诸如 recvFrom 之类的调用来接收数据和有关发送它的机器的信息，用于发送回响应：

```haskell
-- file: ch27/syslogserver.hs
import Data.Bits
import Network.Socket
import Network.BSD
import Data.List
type HandlerFunc = SockAddr -> String -> IO ()
serveLog :: String              -- ^ Port number or name; 514 is default
         -> HandlerFunc         -- ^ Function to handle incoming messages
         -> IO ()
serveLog port handlerfunc = withSocketsDo $
    do -- Look up the port.  Either raises an exception or returns
       -- a nonempty list.
       addrinfos <- getAddrInfo
                    (Just (defaultHints {addrFlags = [AI_PASSIVE]}))
                    Nothing (Just port)
       let serveraddr = head addrinfos
       -- Create a socket
       sock <- socket (addrFamily serveraddr) Datagram defaultProtocol
       -- Bind it to the address we're listening to
       bindSocket sock (addrAddress serveraddr)
       -- Loop forever processing incoming data.  Ctrl-C to abort.
       procMessages sock
    where procMessages sock =
              do -- Receive one UDP packet, maximum length 1024 bytes,
                 -- and save its content into msg and its source
                 -- IP and port into addr
                 (msg, _, addr) <- recvFrom sock 1024
                 -- Handle it
                 handlerfunc addr msg
                 -- And process more messages
                 procMessages sock
-- A simple handler that prints incoming packets
plainHandler :: HandlerFunc
plainHandler addr msg =
    putStrLn $ "From " ++ show addr ++ ": " ++ msg
```

你可以在 ghci 中运行它。调用 `serveLog "1514" plainHandler` 将设置一个 UDP 服务器在端口 1514 上，它将使用 plainHandler 打印该端口上每个传入的 UDP 数据包。Ctrl-C 将终止程序。

## 出现问题时

测试时遇到 `bind: permission denied`？确保你使用的端口号大于 1024。一些操作系统只允许 root 用户绑定到小于 1024 的端口。

## 使用 TCP 通信

TCP 被设计为通过 Internet 尽可能可靠地传输数据。TCP 流量是数据流。虽然这个流会被操作系统分解成单独的数据包，但数据包边界既不知道也与应用无关。TCP 保证，如果流量被传递给应用，它会完整、不修改、恰好一次、按顺序到达。显然，诸如断线之类的事情可能导致流量不被传递，没有协议可以克服这些限制。

与 UDP 相比，这带来了一些权衡。首先，在 TCP 对话开始时必须发送一些数据包来建立链接。因此，对于非常短的对话，UDP 将具有性能优势。此外，TCP 非常努力地让数据通过。如果对话的一方尝试向远程发送数据但没有收到确认，它将定期重传数据一段时间然后放弃。这使得 TCP 在面对丢包时很健壮。然而，这也意味着 TCP 不是涉及实时协议（如直播音频或视频）的最佳选择。

## 处理多个 TCP 流

使用 TCP，连接是有状态的。这意味着在客户端和服务器之间有一个专用的逻辑"通道"，而不是像 UDP 那样的一次性数据包。这让客户端开发变得容易。服务器应用几乎总是希望能够一次处理多个 TCP 连接。那么如何做到这一点呢？

在服务器端，你首先会创建一个套接字并绑定到端口，就像使用 UDP 一样。你的主循环将围绕 accept 调用，而不是重复监听来自任何位置的数据。每次客户端连接时，服务器的操作系统会为其分配一个新的套接字。所以我们有一个主套接字，只用于监听传入连接，从不用于传输数据。我们还有多个子套接字同时使用的可能性，每个对应一个逻辑 TCP 对话。

在 Haskell 中，你通常会使用 forkIO 创建一个单独的轻量级线程来处理每个对话。Haskell 有一个高效的内部实现，性能相当好。

## TCP Syslog 服务器

假设我们要使用 TCP 而不是 UDP 重新实现 syslog。我们可以说单个消息不是由单个数据包定义，而是由尾随换行符 '\n' 定义。任何给定客户端可以使用给定 TCP 连接向服务器发送零个或多个消息。以下是我们如何编写：

```haskell
-- file: ch27/syslogtcpserver.hs
import Data.Bits
import Network.Socket
import Network.BSD
import Data.List
import Control.Concurrent
import Control.Concurrent.MVar
import System.IO
type HandlerFunc = SockAddr -> String -> IO ()
serveLog :: String              -- ^ Port number or name; 514 is default
         -> HandlerFunc         -- ^ Function to handle incoming messages
         -> IO ()
serveLog port handlerfunc = withSocketsDo $
    do -- Look up the port.  Either raises an exception or returns
       -- a nonempty list.
       addrinfos <- getAddrInfo
                    (Just (defaultHints {addrFlags = [AI_PASSIVE]}))
                    Nothing (Just port)
       let serveraddr = head addrinfos
       -- Create a socket
       sock <- socket (addrFamily serveraddr) Stream defaultProtocol
       -- Bind it to the address we're listening to
       bindSocket sock (addrAddress serveraddr)
       -- Start listening for connection requests.  Maximum queue size
       -- of 5 connection requests waiting to be accepted.
       listen sock 5
       -- Create a lock to use for synchronizing access to the handler
       lock <- newMVar ()
       -- Loop forever waiting for connections.  Ctrl-C to abort.
       procRequests lock sock
    where
          -- | Process incoming connection requests
          procRequests :: MVar () -> Socket -> IO ()
          procRequests lock mastersock =
              do (connsock, clientaddr) <- accept mastersock
                 handle lock clientaddr
                    "syslogtcpserver.hs: client connnected"
                 forkIO $ procMessages lock connsock clientaddr
                 procRequests lock mastersock
          -- | Process incoming messages
          procMessages :: MVar () -> Socket -> SockAddr -> IO ()
          procMessages lock connsock clientaddr =
              do connhdl <- socketToHandle connsock ReadMode
                 hSetBuffering connhdl LineBuffering
                 messages <- hGetContents connhdl
                 mapM_ (handle lock clientaddr) (lines messages)
                 hClose connhdl
                 handle lock clientaddr
                    "syslogtcpserver.hs: client disconnected"
          -- Lock the handler before passing data to it.
          handle :: MVar () -> HandlerFunc
          -- This type is the same as
          -- handle :: MVar () -> SockAddr -> String -> IO ()
          handle lock clientaddr msg =
              withMVar lock
                 (\a -> handlerfunc clientaddr msg >> return a)
-- A simple handler that prints incoming packets
plainHandler :: HandlerFunc
plainHandler addr msg =
    putStrLn $ "From " ++ show addr ++ ": " ++ msg
```

我们的 SyslogTypes 实现请参阅"UDP 客户端示例：syslog"部分。

让我们看看这段代码。我们的主循环在 procRequests 中，我们在那里循环等待来自客户端的新连接。accept 调用会阻塞直到客户端连接。当客户端连接时，我们获得一个新的套接字和客户端的地址。我们将一条关于该消息的消息传递给处理程序，然后使用 forkIO 创建一个线程来处理该客户端的数据。此线程运行 procMessages。

在处理 TCP 数据时，将套接字转换为 Haskell Handle 通常很方便。我们在这里这样做，并明确设置缓冲——这是 TCP 通信的一个重要点。接下来，我们从套接字的 Handle 设置惰性读取。对于每个传入的行，我们将其传递给 handle。当没有更多数据时——因为远程端已关闭套接字——我们输出一条关于该消息的消息。

由于我们可能同时处理多个传入消息，我们需要确保不会在处理程序中一次输出多条消息。这可能导致输出乱码。我们使用一个简单的锁来序列化对处理程序的访问，并编写一个简单的 handle 函数来处理。

## TCP Syslog 客户端

现在，让我们为我们的 TCP syslog 协议编写一个客户端。此客户端将类似于 UDP 客户端，但有一些变化。首先，由于 TCP 是流协议，我们可以使用 Handle 发送数据，而不是使用较低级的套接字操作。其次，我们不再需要在 SyslogHandle 中存储目标地址，因为我们将要使用 connect 来建立 TCP 连接。最后，我们需要一种方法来知道一条消息在哪里结束以及下一条在哪里开始。对于 UDP，这很容易，因为每条消息都是一个独立的逻辑数据包。对于 TCP，我们将只使用换行符 '\n' 作为消息结束标记，尽管这意味着任何单独的消息都不能包含换行符。这是我们的代码：

```haskell
-- file: ch27/syslogtcpclient.hs
import Data.Bits
import Network.Socket
import Network.BSD
import Data.List
import SyslogTypes
import System.IO
data SyslogHandle =
    SyslogHandle {slHandle :: Handle,
                  slProgram :: String}
openlog :: HostName             -- ^ Remote hostname, or localhost
        -> String               -- ^ Port number or name; 514 is default
        -> String               -- ^ Name to log under
        -> IO SyslogHandle      -- ^ Handle to use for logging
openlog hostname port progname =
    do -- Look up the hostname and port.  Either raises an exception
       -- or returns a nonempty list.  First element in that list
       -- is supposed to be the best option.
       addrinfos <- getAddrInfo Nothing (Just hostname) (Just port)
       let serveraddr = head addrinfos
       -- Establish a socket for communication
       sock <- socket (addrFamily serveraddr) Stream defaultProtocol
       -- Mark the socket for keep-alive handling since it may be idle
       -- for long periods of time
       setSocketOption sock KeepAlive 1
       -- Connect to server
       connect sock (addrAddress serveraddr)
       -- Make a Handle out of it for convenience
       h <- socketToHandle sock WriteMode
       -- We're going to set buffering to BlockBuffering and then
       -- explicitly call hFlush after each message, below, so that
       -- messages get logged immediately
       hSetBuffering h (BlockBuffering Nothing)

       -- Save off the socket, program name, and server address in a handle
       return $ SyslogHandle h progname
syslog :: SyslogHandle -> Facility -> Priority -> String -> IO ()
syslog syslogh fac pri msg =
    do hPutStrLn (slHandle syslogh) sendmsg
       -- Make sure that we send data immediately
       hFlush (slHandle syslogh)
    where code = makeCode fac pri
          sendmsg = "<" ++ show code ++ ">" ++ (slProgram syslogh) ++
                    ": " ++ msg
closelog :: SyslogHandle -> IO ()
closelog syslogh = hClose (slHandle syslogh)
{- | Convert a facility and a priority into a syslog code -}
makeCode :: Facility -> Priority -> Int
makeCode fac pri =
    let faccode = codeOfFac fac
        pricode = fromEnum pri
        in
          (faccode `shiftL` 3) .|. pricode
```
