# CIS 194 作业 3 — 代码高尔夫（Code Golf）

**截止日期：2月4日，星期一**

---

## 概述

这个作业很简单：下面有三个任务。对于每个任务，你需要提交一个具有指定名称和类型签名的 Haskell 函数来完成给定任务，并且要尽可能短。

---

## 规则

- **必须包含注释**：每个任务的解决方案必须包含解释你的解决方案及其工作原理的注释。没有解释性注释的解决方案将得零分。你的注释应该展示你对解决方案的完整理解。
- **注释不计入长度**
- **类型签名不计入长度**
- **import 语句不计入长度**：你可以导入 Haskell Platform 包含的任何模块
- **空白不计入长度**：不需要把所有代码塞到一行，适当使用空格和缩进，但尽量让代码更短
- **可以包含额外函数**：欢迎在需要时添加额外的函数来分解解决方案（有时候这会让你得到更短的解法）。当然，这些额外函数会计入解决方案的长度（不包括其类型签名）
- **提交文件名**：`Golf.hs`，文件应定义一个名为 `Golf` 的模块，即文件顶部应该有 `module Golf where`
- **代码高尔夫积分**：每个任务最短的三个解决方案（计算总字符数，不包括空白和其他例外情况）可获得 2 分额外积分。最多可以获得 4 分额外积分
- **否则，长度并不重要**：长但正确的解决方案在正确性方面会获得满分（不过在风格方面可能会或不会获得满分，取决于风格）

---

## 提示

- **尽可能使用标准库函数**：这是作业的要点。使用 `map` 比自己实现要短得多！
- **使用封装了递归模式的函数**：特别是，尝试使用标准库中封装了递归模式的函数，而不是自己写显式递归函数
- **先实现，后优化**：你可以先写出能工作的版本，不担心长度。一旦你解决了任务，再想办法让你的解决方案更短
- **如果有疑问**：可以在 Piazza 上提问
- **测试你的函数**：我们会在示例之外的输入上测试你的函数，所以请确保你的函数是安全的！

## 练习 1：跳房子（Hopscotch）

### 任务

编写一个函数：

```haskell
skips :: [a] -> [[a]]
```

### 说明

`skips` 的输出是一个列表的列表。输出列表的长度应与输入列表相同。

- 第一个列表与输入列表相同
- 第二个列表包含输入列表中的每个第二个元素
- ……以此类推，第 n 个列表包含输入列表中的每个第 n 个元素

### 示例

```haskell
skips "ABCD"       == ["ABCD", "BD", "C", "D"]
skips "hello!"     == ["hello!", "el!", "l!", "l", "o", "!"]
skips [1]          == [[1]]
skips [True, False] == [[True, False], [False]]
skips []           == []
```

注意：输出长度应该与输入长度相同。

## 练习 2：局部最大值（Local Maxima）

### 任务

编写一个函数：

```haskell
localMaxima :: [Integer] -> [Integer]
```

### 说明

列表的**局部最大值**是严格大于其前后元素的元素。

例如，在列表 `[2, 3, 4, 1, 5]` 中：
- `4` 是局部最大值（因为它大于前一个元素 3 和后一个元素 1）
- `5` **不是**局部最大值（因为它后面没有元素）

### 示例

```haskell
localMaxima [2, 9, 5, 6, 1] == [9, 6]
localMaxima [2, 3, 4, 1, 5] == [4]
localMaxima [1, 2, 3, 4, 5] == []
```

## 练习 3：直方图（Histogram）

### 任务

编写一个函数：

```haskell
histogram :: [Integer] -> String
```

### 说明

输入是 0 到 9 之间的整数列表（包含端点），输出是一个垂直直方图，显示每个数字在输入列表中出现的次数。

你可以假设输入列表不包含小于 0 或大于 9 的数字（即如果输入包含这样的数字，你的函数做什么都可以）。

你的输出必须与下面示例中显示的输出完全匹配。

### 示例

```haskell
-- 输入 [1,1,1,5] 的输出：
histogram [1,1,1,5] ==
    "   *   \n    *   \n    *   \n    *   \n==========\n0123456789\n"

-- 实际显示效果：
   *
   *
   *       *
==========
0123456789

-- 输入 [1,4,5,4,6,6,3,4,2,4,9] 的输出：
histogram [1,4,5,4,6,6,3,4,2,4,9] ==
    " *        \n *   **   *\n==========\n0123456789\n"

-- 实际显示效果：

```

### 重要提示

在 ghci 中输入类似 `histogram [3,5]` 的内容时，你应该会看到类似这样的字符串表示：

```
"   *\n *   *\n==========\n0123456789\n"
```

这是 String 输出的文本表示，包括表示换行符的 `\n` 转义序列。要实际可视化如上例所示的直方图，请使用 `putStr`，例如：

```haskell
putStr (histogram [3,5])
```
