## if 语句

根据你使用的语言，你可能可以使用几种 if 语句。最简单的是普通 if 或 if-then 语句。if-then-else 稍微复杂一些，而 if-then-else-if 链是最复杂的。

---

## 普通 if-then 语句

编写 if 语句时遵循这些指导原则：

**首先编写代码的正常路径；然后编写异常情况**。编写代码使代码的正常路径清晰。确保罕见情况不会掩盖执行的正常路径。这对可读性和性能都很重要。

**确保在相等性上正确分支**。使用 `>` 而不是 `>=`，或 `<` 而不是 `<=`，类似于在访问数组或计算循环索引时犯越界错误。在循环中，考虑端点以避免越界错误。在条件语句中，考虑相等情况以避免错误。

**将正常情况放在 if 之后而不是 else 之后**。首先处理你通常期望的情况。这与将决策结果代码尽可能靠近决策的一般原则一致。以下是一个在过程中随意检查错误的代码示例：

**考虑 else 子句**。如果你认为你需要一个普通 if 语句，考虑你是否实际上需要一个 if-then-else 语句。一项经典的通用汽车分析发现，50% 到 80% 的 if 语句应该有 else 子句（Elshoff 1976）。

```java
   ...
}
else {
   // else color is invalid
   // screen not written to --- safely ignore command
}
```

**检查 if 和 else 子句是否反转**。编程 if-then 时的一个常见错误是颠倒了应该跟随 if 子句的代码和应该跟随 else 子句的代码，或者将 if 测试的逻辑弄反了。检查你的代码是否有这个常见错误。

## if-then-else 语句链

在不支持 case 语句的语言中——或者只部分支持——你经常会发现自己在编写 if-then-else 测试链。例如，对字符进行分类的代码可能使用像这样的链：

示例 15-6. 使用 if-then-else 链对字符进行分类的 C++ 示例

```cpp
if ( inputCharacter < SPACE ) {
   characterType = CharacterType_ControlCharacter;
}
else if (
   inputCharacter == ' ' ||
   inputCharacter == ',' ||
   inputCharacter == '.' ||
   inputCharacter == '!' ||
   inputCharacter == '(' ||
   inputCharacter == ')' ||
   inputCharacter == ':' ||
   inputCharacter == ';' ||
   inputCharacter == '?' ||
   inputCharacter == '-'
   ) {
   characterType = CharacterType_Punctuation;
}
else if ( '0' <= inputCharacter && inputCharacter <= '9' ) {
   characterType = CharacterType_Digit;
}
else if (
   ( 'a' <= inputCharacter && inputCharacter <= 'z' ) ||
   ( 'A' <= inputCharacter && inputCharacter <= 'Z' )
   ) {
   characterType = CharacterType_Letter;
}
```

关于简化复杂表达式的更多细节，请参见"布尔表达式"。

编写这种 if-then-else 链时考虑这些指导原则：

**用布尔函数调用简化复杂测试**。上一个例子代码难以阅读的原因之一是对字符进行分类的测试很复杂。为了提高可读性，你可以用布尔函数调用替换它们。以下是测试被替换为布尔函数后例子代码的样子：

示例 15-7. 使用布尔函数调用的 if-then-else 链的 C++ 示例

```cpp
if ( IsControl( inputCharacter ) ) {
   characterType = CharacterType_ControlCharacter;
}
else if ( IsPunctuation( inputCharacter ) ) {
   characterType = CharacterType_Punctuation;
}
else if ( IsDigit( inputCharacter ) ) {
   characterType = CharacterType_Digit;
}
else if ( IsLetter( inputCharacter ) ) {
   characterType = CharacterType_Letter;
}
```

**将最常见的情况放在前面**。通过将最常见的情况放在前面，你最小化了某人必须阅读的异常情况处理代码量以找到通常情况。你提高了效率，因为你最小化了代码为找到最常见情况而执行的测试次数。在刚才显示的例子中，字母比标点符号更常见，但标点符号的测试首先进行。以下是修改后的代码，以便首先测试字母：

示例 15-8. 首先测试最常见情况的 C++ 示例

```cpp
if ( IsLetter( inputCharacter ) ) {       // 1
   characterType = CharacterType_Letter;
}
else if ( IsPunctuation( inputCharacter ) ) {
   characterType = CharacterType_Punctuation;
}
else if ( IsDigit( inputCharacter ) ) {
   characterType = CharacterType_Digit;
}
else if ( IsControl( inputCharacter ) ) {       // 2
   characterType = CharacterType_ControlCharacter;
}
```

(1) 这个测试是最常见的，现在首先执行。

(2) 这个测试是最不常见的，现在最后执行。

**确保覆盖所有情况**。用错误消息或断言编码最后一个 else 子句以捕获你没有计划的情况。这个错误消息是为你而不是用户准备的，所以要适当措辞。以下是你如何修改字符分类例子来执行"其他情况"测试：

示例 15-9. 使用默认情况捕获错误的 C++ 示例

```cpp

}
else if ( IsPunctuation( inputCharacter ) ) {
   characterType = CharacterType_Punctuation;
}
```

## case 语句

case 或 switch 语句是一种因语言不同而有很大差异的结构。C++ 和 Java 仅支持一次取一个值的有序类型 case。Visual Basic 支持有序类型的 case，并有用于表达范围和值组合的强大简写表示法。许多脚本语言根本不支持 case 语句。

以下各节提供有效使用 case 语句的指导原则：

---

## 选择最有效的 case 顺序

你可以选择多种方式来组织 case 语句中的 case。如果你有一个只有三个选项和三行相应代码的小型 case 语句，你使用的顺序不太重要。如果你有一个长 case 语句——例如，在事件驱动程序中处理数十个事件的 case 语句——顺序很重要。以下是一些排序可能性：

**按字母或数字顺序排列 case**。如果 case 同等重要，按 A-B-C 顺序放置它们会提高可读性。这样，特定的 case 很容易从组中挑选出来。

**将正常情况放在前面**。如果你有一个正常情况和几个例外，将正常情况放在前面。用注释表明这是正常情况，其他是异常的。

**按频率排序 case**。将最频繁执行的 case 放在前面，最不频繁执行的放在最后。这种方法有两个优势。首先，人类读者可以轻松找到最常见的情况。扫描列表寻找特定 case 的读者可能对最常见的情况之一感兴趣，将常见情况放在代码顶部使搜索更快。

---

## 使用 case 语句的技巧

以下是使用 case 语句的几个技巧：

**保持每个 case 的动作简单**。保持与每个 case 关联的代码简短。每个 case 后的简短代码有助于使 case 语句的结构清晰。如果为 case 执行的动作很复杂，编写一个例程并从 case 调用该例程，而不是将代码放入 case 本身。

交叉参考

关于简化代码的其他技巧，请参见第24章。

**不要编造虚假变量来使用 case 语句**。case 语句应该用于容易分类的简单数据。如果你的数据不简单，使用 if-then-else 链代替。虚假变量是令人困惑的，你应该避免。例如，不要这样做：

**创建虚假的 case 变量的 Java 示例——糟糕的做法**

示例 15-11.

```java
action = userCommand[ 0 ];
switch ( action ) {
   case 'c':
      Copy();
      break;
   case 'd':
      DeleteCharacter();
      break;
   case 'f':
      Format();
      break;
   case 'h':
      Help();
      break;

   ...
   default:
      HandleUserInputError( ErrorType.InvalidUserCommand );
}
```

控制 case 语句的变量是 `action`。在这种情况下，`action` 是通过剥离 `userCommand` 字符串的第一个字符创建的，这是一个用户输入的字符串。

这个制造麻烦的代码来自错误的一侧，会招致问题。一般来说，当你为 case 语句制造一个变量时，真实数据可能不会按你想要的方式映射到 case 语句。在这个例子中，如果用户输入 copy，case 语句剥离第一个"c"并正确调用 `Copy()` 例程。另一方面，如果用户输入 cement、overshoes、clambake 或 cellulose，case 语句也会剥离"c"并调用 `Copy()`。case 语句 else 子句中对错误命令的测试不会很好地工作，因为它只会错过错误的首字母而不是错误的命令。

交叉参考

不要编造虚假变量，这段代码应该使用 if-then-else-if 链来检查整个字符串。代码的Virtuous重写如下：


**仅将 default 子句用于检测合法默认值**。你有时可能只有一个 case 剩余，并决定将该 case 编码为 default 子句。虽然有时很诱人，但这是愚蠢的。你会失去 case 语句标签提供的自动文档，以及用 default 子句检测错误的能力。

这样的 case 语句在修改下会崩溃。如果你使用合法的 default，添加新 case 是微不足道的——你只需添加 case 和相应的代码。如果你使用虚假的 default，修改会更困难。你必须添加新 case，可能使其成为新的 default，然后更改以前用作 default 的 case，使其成为合法的 case。首先使用合法的 default。

**使用 default 子句检测错误**。如果 case 语句中的 default 子句不用于其他处理且不应该发生，在其中放置诊断消息：

示例 15-13. 使用默认情况检测错误的 Java 示例——好的做法

```java
switch ( commandShortcutLetter ) {
   case 'a':
      PrintAnnualReport();
      break;
   case 'p':
      // no action required, but case was considered
      break;
   case 'q':
      PrintQuarterlyReport();
      break;
   case 's':
      PrintSummaryReport();
      break;
   default:
      DisplayInternalError( "Internal Error 905: Call customer support." );
}
```

像这样的消息在调试和生产代码中都有用。大多数用户更喜欢"内部错误：请致电客户支持"这样的消息，而不是系统崩溃，或者更糟的是，看起来正确但直到用户的老板检查时才出错的微妙错误结果。

如果 default 子句用于错误检测以外的其他目的，含义是每个 case 选择器都是正确的。仔细检查以确保每个可能进入 case 语句的值都是合法的。如果你发现有些不是合法的，重写语句以便 default 子句将检查错误。

**在 C++ 和 Java 中，避免在 case 语句末尾落空**。C 类语言（C、C++ 和 Java）不会自动从每个 case 跳出。相反，你必须在每个 case 末尾明确编码。如果你不在 case 末尾编码，程序会落空并执行下一个 case 的代码。这可能导致一些特别恶劣的编码实践，包括以下可怕的例子：

**滥用 case 语句的 C++ 示例**

示例 15-14.

```cpp
switch ( InputVar ) {
   case 'A': if ( test ) {
                   // statement 1
                   // statement 2
   case 'B':       // statement 3
                   // statement 4
                   ...
               }
               ...
            break;
   ...
}
```

这种做法是坏的，因为它混合了控制构造。嵌套控制构造已经够难理解了；重叠的构造几乎不可能。修改 case 'A' 或 case 'B' 将比脑外科手术更难，而且在任何修改工作之前，可能需要清理这些 case。你最好第一次就做对。一般来说，避免在 case 语句末尾落空是个好主意。

**在 C++ 中，清楚明确地识别 case 语句末尾的直通**。如果你故意编写在 case 末尾落空的代码，在发生的地方清楚注释并解释为什么需要那样编码。

示例 15-15. 文档化 case 语句末尾落空的 C++ 示例

```cpp
switch ( errorDocumentationLevel ) {
   case DocumentationLevel_Full:
      DisplayErrorDetails( errorNumber );
      // FALLTHROUGH -- Full documentation also prints summary comments

   case DocumentationLevel_Summary:
      DisplayErrorSummary( errorNumber );
      // FALLTHROUGH -- Summary documentation also prints error number

   case DocumentationLevel_NumberOnly:
      DisplayErrorNumber( errorNumber );
      break;

   default:
      DisplayInternalError( "Internal Error 905: Call customer support." );
}
```

这种技术的用处就像你发现有人宁愿要一辆用过的庞蒂亚克 Aztek 也不要一辆新的科尔维特一样。一般来说，从一个 case 落入另一个 case 的代码是代码被修改时招致错误的邀请，应该避免。

## 关键点

- 对于简单的 if-else 语句，注意 if 和 else 子句的顺序，特别是在处理大量错误时。确保正常情况清晰。
- 对于 if-then-else 链和 case 语句，选择最大化可读性的顺序。
- 要捕获错误，在 case 语句中使用 default 子句，或在 if-then-else 语句链中的最后一个 else 中使用。
- 并非所有控制构造都是平等的。为代码的每个部分选择最合适的控制构造。
