# Haskell 标准库函数

## 概述

Haskell 的标准库（Prelude）提供了大量实用的内置函数。作业中明确要求使用以下函数：**`read`**、`lines`、`words`、`unwords`、`take`、`drop` 和函数复合（`.`）。善用这些函数可以写出简洁、优雅的代码——**不要重复造轮子**。

---

## read — 字符串转值

```haskell
read :: Read a => String -> a
```

将字符串**解析**为指定类型的值。必须通过类型签名或上下文**显式指定目标类型**。

### 基本示例

```haskell
ghci> read "42" :: Int
42

ghci> read "3.14" :: Double
3.14

ghci> read "[1,2,3]" :: [Int]
[1,2,3]

ghci> read "True" :: Bool
True
```

### 为什么需要类型注解

```haskell
ghci> read "42"
-- *** Exception: Prelude.read: no parse
-- Haskell 不知道"42"应该变成 Int 还是 Double 还是其他类型

ghci> read "42" :: Int     -- ✓ 明确告诉编译器
42

ghci> read "42" :: Double  -- ✓ 结果完全不同
42.0
```

### 作业中的应用

解析日志中的数字字段：

```haskell
case words "E 2 562 help help" of
    ('E':sev:ts:rest) ->
        LogMessage (Error (read sev)) (read ts) (unwords rest)
    --              ↑                    ↑
    --        "2"   → 2           "562" → 562
```

## words — 按空白字符分割

```haskell
words :: String -> [String]
```

将字符串按空白字符（空格、Tab、换行）分割为单词列表。

### 基本示例

```haskell
ghci> words "hello world"
["hello", "world"]

ghci> words "hello   world\nfoo"
["hello", "world", "foo"]

ghci> words "  skip  spaces  "
["skip", "spaces"]

ghci> words ""
[]
```

### 作业中的应用

解析日志行的第一步——将一行字符串拆分为各个字段：

```haskell
words "E 2 562 help help"
  == ["E", "2", "562", "help", "help"]
```

然后用模式匹配提取各部分：

```haskell
case words s of
    ('E':sev:ts:rest) -> ...  -- sev="2", ts="562", rest=["help","help"]
    ('I':ts:rest)     -> ...
    _                  -> Unknown s
```

---

## unwords — 合并为字符串

```haskell
unwords :: [String] -> String
```

将字符串列表用**单个空格**连接为一行。是 `words` 的逆操作。

### 基本示例

```haskell
ghci> unwords ["hello", "world"]
"hello world"

ghci> unwords ["a", "b", "c"]
"a b c"

ghci> unwords []
""
```

### 作业中的应用

将剩余单词合并回字符串：

```haskell
case words "E 2 562 help help" of
    ('E':sev:ts:rest) ->
        LogMessage (Error (read sev)) (read ts) (unwords rest)
    --                                         ↑
    --                              ["help", "help"] → "help help"
```

### `words` 与 `unwords` 的配对使用

```haskell
ghci> unwords (words "hello   world\nfoo")
"hello world foo"
-- 先分割（合并多个空白），再连接
```

## lines — 按换行符分割

```haskell
lines :: String -> [String]
```

将字符串按 `\n` 分割为行列表。

### 基本示例

```haskell
ghci> lines "hello\nworld\nfoo"
["hello", "world", "foo"]

ghci> lines "line1\nline2\n"
["line1", "line2"]

ghci> lines ""
[]
```

### 配合 `unlines` 使用

```haskell
ghci> unlines ["a", "b", "c"]
"a\nb\nc\n"
```

---

## take — 取前 n 个元素

```haskell
take :: Int -> [a] -> [a]
```

从列表**开头**取 n 个元素。

### 基本示例

```haskell
ghci> take 3 [1,2,3,4,5]
[1,2,3]

ghci> take 10 [1,2,3]      -- 不够就返回全部
[1,2,3]

ghci> take 0 [1,2,3]
[]

ghci> take (-1) [1,2,3]    -- n <= 0 返回空列表
[]

ghci> take 2 "hello"
"he"
```

### 常用场景

```haskell
-- 取整数的前几位数字
ghci> take 3 [1,2,3,4,5]
[1,2,3]

-- 与函数复合
ghci> take 5 (lines content)  -- 取文件前 5 行
```

---

## drop — 丢弃前 n 个元素

```haskell
drop :: Int -> [a] -> [a]
```

从列表**开头**丢弃 n 个元素，返回剩余部分。

### 基本示例

```haskell
ghci> drop 2 [1,2,3,4,5]
[3,4,5]

ghci> drop 10 [1,2,3]       -- 丢弃超过长度的数量
[]

ghci> drop 0 [1,2,3]
[1,2,3]

ghci> drop (-1) [1,2,3]    -- n < 0 返回原列表
[1,2,3]

ghci> drop 2 "hello"
"llo"
```

### `take` 和 `drop` 的关系

```haskell
take n xs ++ drop n xs == xs   -- 任意 n，两者合并等于原列表
```

## 函数复合（`.`）— 组合多个函数

```haskell
(.) :: (b -> c) -> (a -> b) -> a -> c
(f . g) x = f (g x)
```

将两个函数**串联**成新函数：**先执行 g，再把结果传给 f**。

### 基本示例

```haskell
ghci> (+1) . (*2) $ 5
11          -- (5 * 2) + 1 = 11

ghci> even . length $ "hello"
False       -- 先算长度 5，再判断是否为偶数

ghci> (head . words) "hello world"
"hello"     -- 先分词，再取第一个
```

### 作业中的应用

```haskell
-- 解析整个文件：先按行分割，再逐行解析
parse :: String -> [LogMessage]
parse = map parseMessage . lines
--              ↑              ↑
--         先用 lines 按行分割
--                        再对每行应用 parseMessage
```

等价于：

```haskell
parse s = map parseMessage (lines s)
```

### 函数复合的链式写法

```haskell
-- 读取文件 → 按行分割 → 取前 10 行 → 解析
process :: String -> [LogMessage]
process = map parseMessage . take 10 . lines

-- 等价于：
process s = map parseMessage (take 10 (lines s))
```

### 何时用函数复合 vs 显式参数

```haskell
-- 函数复合：适合直接传递结果的场景
parse = map parseMessage . lines

-- 显式参数：适合中间结果需要复用的场景
parse s = let lines' = lines s
               in map parseMessage lines'
```

## 综合示例：日志解析

将所有函数组合起来，解析日志文件：

```haskell
-- 完整流程
parse :: String -> [LogMessage]
parse = map parseMessage . lines

parseMessage :: String -> LogMessage
parseMessage s = case words s of
    ('E':sev:ts:rest) -> LogMessage (Error (read sev)) (read ts) (unwords rest)
    ('I':ts:rest)     -> LogMessage Info (read ts) (unwords rest)
    ('W':ts:rest)     -> LogMessage Warning (read ts) (unwords rest)
    _                  -> Unknown s

-- 解析前 5 行
testParse :: Int -> String -> [LogMessage]
testParse n = take n . parse
--           ↑           ↑
--        取前 n 个   先解析
```

执行流程图：

```
"E 2 562 help\nI 29 la"
        ↓
   lines
        ↓
["E 2 562 help", "I 29 la"]
        ↓
   words
        ↓
[["E","2","562","help"], ["I","29","la"]]
        ↓
   read + unwords
        ↓
[LogMessage (Error 2) 562 "help", LogMessage Info 29 "la"]
```

## 常用标准库速查表

| 函数 | 类型 | 作用 |
|------|------|------|
| `words` | `String -> [String]` | 按空白分割 |
| `unwords` | `[String] -> String` | 空格连接 |
| `lines` | `String -> [String]` | 按换行分割 |
| `unlines` | `[String] -> String` | 换行连接 |
| `take` | `Int -> [a] -> [a]` | 取前 n 个 |
| `drop` | `Int -> [a] -> [a]` | 丢弃前 n 个 |
| `head` | `[a] -> a` | 取第一个元素 |
| `tail` | `[a] -> [a]` | 去掉第一个 |
| `last` | `[a] -> a` | 取最后一个 |
| `init` | `[a] -> [a]` | 去掉最后一个 |
| `null` | `[a] -> Bool` | 是否为空列表 |
| `length` | `[a] -> Int` | 列表长度 |
| `read` | `Read a => String -> a` | 字符串转值 |
| `show` | `Show a => a -> String` | 值转字符串 |
| `(.)` | `(b -> c) -> (a -> b) -> a -> c` | 函数复合 |
