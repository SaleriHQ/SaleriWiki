# 全函数与偏函数 (Total & Partial Functions)

## 定义

| 术语 | 定义 | Haskell 例子 |
|------|------|-------------|
| **全函数 (Total Function)** | 对所有合法输入都有定义的函数 | `safeHead :: [a] -> Maybe a` |
| **偏函数 (Partial Function)** | 存在某些输入导致崩溃或无限循环 | `head :: [a] -> a`（空列表崩溃） |

## 偏函数的危害

`head` 的类型签名 `[a] -> a` 声称"给我一个列表，我返回一个同类型的值"。但实际上空列表会崩溃：

```haskell
head []  -- 💥 "Prelude.head: empty list"
```

这是**类型系统撒谎**：类型承诺的功能对某些输入无法兑现。

## Haskell 的解决方案

### 1. 用 `Maybe` 编码失败可能性

```haskell
-- 偏函数（会崩溃）
head :: [a] -> a

-- 全函数（用类型反映可能的失败）
safeHead :: [a] -> Maybe a
safeHead []    = Nothing
safeHead (x:_) = Just x
```

使用 `safeHead` 的调用者**必须**处理 `Nothing` 情况，编译器强制检查：

```haskell
process :: [a] -> String
process xs = case safeHead xs of
  Nothing  -> "列表为空"
  Just x   -> "第一个元素是: " ++ show x
```

### 2. 用 `NonEmptyList` 编码非空保证

当语义上保证列表非空时，用新类型让编译器执行：

```haskell
data NonEmptyList a = NEL a [a]

nelHead :: NonEmptyList a -> a
nelHead (NEL a _) = a  -- 永远安全，无需 Maybe
```

## 应避免的偏函数

来自 [[wiki/sources/cis194-lecture3-rec-poly]] 的警告：

> **`head` 是错误！** `tail`、`init`、`last`、`(!!)` 都应避免使用。

| 偏函数 | 全函数替代 |
|--------|-----------|
| `head` | `safeHead :: [a] -> Maybe a` |
| `tail` | `viaMaybe tail :: [a] -> Maybe [a]` |
| `init` | `initSafe :: [a] -> Maybe [a]` |
| `last` | `safeLast :: [a] -> Maybe a` |
| `(!!)` | `atMay :: [a] -> Int -> Maybe a` |

（在 `safe` 包中有完整实现：`hackage.haskell.org/package/safe`）

## 核心原则

> **类型系统应尽可能多地反映函数的实际行为。**

偏函数的危害不只存在于 Haskell——在任何语言中，用运行时异常表示可预见的失败都是糟糕的设计。Haskell 的优势在于，`Maybe` 类型让类型系统可以**静态地**强制处理失败情况。
