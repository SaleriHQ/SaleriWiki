# 类型类 (Type Class)

## 定义

类型类是 Haskell 实现**特设多态**（ad-hoc polymorphism）的机制。它类似于其他语言中的"接口"——定义一组操作（函数），凡是想使用这些操作的类型都必须提供这些操作的实现。

> 类比理解：类型类 ≈ 生物学中的"界门纲目科属种"分类系统。每个类群定义了共同的特征，物种（类型）属于某个类群后必须具有这些特征。

## 核心类型类

### Eq — 相等性

```haskell
class Eq a where
  (==) :: a -> a -> Bool
  (/=) :: a -> a -> Bool
```

支持 `==` 和 `/=` 的类型：

```haskell
ghci> 3 == 3          -- True
ghci> "hello" /= "hi"  -- True
ghci> [1,2] == [1,2]   -- True
```

### Ord — 可排序

```haskell
class Eq a => Ord a where
  compare :: a -> a -> Ordering
  (<)  :: a -> a -> Bool
  (<=) :: a -> a -> Bool
  (>)  :: a -> a -> Bool
  (>=) :: a -> a -> Bool
  max  :: a -> a -> a
  min  :: a -> a -> a
```

```haskell
ghci> compare 3 5       -- LT
ghci> max 3 5            -- 5
ghci> 'a' < 'b'          -- True
ghci> "apple" < "banana" -- True（字典序）
ghci> [1,2,3] < [1,2,4]  -- True（列表逐元素比较）
```

### Show — 可显示

```haskell
class Show a where
  show :: a -> String
```

```haskell
ghci> show 3           -- "3"
ghci> show True        -- "True"
ghci> show [1,2,3]     -- "[1,2,3]"
```

### Read — 可解析

```haskell
class Read a where
  read :: String -> a
```

```haskell
ghci> read "3" + 4     -- 7（类型推导决定 a = Int）
ghci> read "True" || False  -- True
```

> 注意：单独使用 `read "3"` 会报错，需要提供足够的上下文让 GHC 推断出类型。

### Num — 数值

```haskell
class (Eq a, Show a) => Num a where
  (+) :: a -> a -> a
  (-) :: a -> a -> a
  (*) :: a -> a -> a
  negate :: a -> a
  abs :: a -> a
  signum :: a -> a
  fromInteger :: Integer -> a
```

Num 约束隐含 Eq 和 Show，且自动支持 `fromIntegral` 从 Int 到 Integer 等。

```haskell
ghci> :info (+)
class (Eq a, Show a) => Num a where
  (+) :: a -> a -> a
  ...
-- Num 本身约束 Eq 和 Show，这就是"上下文约束"
```

## 为自定义类型实现类型类

使用 `instance` 关键字：

```haskell
data Color = Red | Green | Blue

-- 让 Color 成为 Eq 类型类的一个实例
instance Eq Color where
  Red   == Red   = True
  Green == Green = True
  Blue  == Blue  = True
  _     == _     = False   -- 其余情况均不相等

-- Ord 实例
instance Ord Color where
  compare Red   Red   = EQ
  compare Red   _     = LT   -- Red 最小
  compare Blue  Blue  = EQ
  compare Blue  _     = GT   -- Blue 最大
  compare Green Green = EQ
```

## 自动派生（deriving）

Haskell 可以用 `deriving` 自动生成常见类型类的实例——只需一行：

```haskell
data Color = Red | Green | Blue
           deriving (Eq, Ord, Show, Read)

-- 等价于手动实现 Eq 和 Show：
ghci> Red == Green          -- False（Eq 自动生成）
ghci> show Red              -- "Red"（Show 自动生成）
ghci> read "Green" :: Color -- Green（Read 自动生成）
```

> 规律：如果类型的构造器组合简单明了，可以直接 `deriving`；如果需要特殊语义，则手动 `instance`。

## 类型类约束（Context）

类型类可以嵌套约束：

```haskell
-- Eq a => Ord a 意味着：要实现 Ord，必须先实现 Eq
class (Eq a) => Ord a where
  compare :: a -> a -> Ordering
  ...

-- 实际函数签名示例
duplicate :: (Eq a, Show a) => a -> String
duplicate x = if x == x then show x else ""
-- 必须同时是 Eq 和 Show
```

## 类型类与多态

类型类是 Haskell 三种多态机制之一：

| 机制 | 关键字 | 特点 | 示例 |
|------|--------|------|------|
| 参数多态 | 多态类型变量 `a` | 所有类型统一实现 | `id :: a -> a` |
| 特设多态 | 类型类约束 | 不同类型不同实现 | `sort :: Ord a => [a] -> [a]` |
| 约束多态 | 子类型（无） | Haskell 无子类型继承 | — |

## 常见标准类型类层次

```
Num
├── Eq ← Ord ← (Num a)
├── Show
└── Read

Enum      — 可枚举（用于 [1..10]）
Bounded   — 有界（minBound, maxBound）
Foldable  — 可折叠
Functor   — 可映射
Applicative — 可应用
Monad     — 单子（进阶）
```
