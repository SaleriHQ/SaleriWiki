## 摘要

伪代码编程过程(PPP)是创建高质量类和例程的系统方法。本章描述构建类和例程的步骤、如何编写有效伪代码、使用PPP的详细过程以及替代方案。核心观点：好的伪代码用类似英语的语句描述特定操作，避免目标语言语法，在意图层次编写；伪代码变成注释，确保注释准确。

## 专业人士的伪代码

**有效伪代码的原则**：
- 使用类似英语的语句精确描述特定操作
- 避免目标编程语言的语法元素
- 在意图层次编写（描述做什么而非如何做）
- 在足够低的层次编写（可自动生成代码）

**好vs坏伪代码示例**：

```cpp
// 坏伪代码
*hRsrcPtr = resource number;
return 1;

// 好伪代码
Keep track of current number of resources in use
If another resource is available
   Allocate a dialog box structure
   Note that one more resource is in use
   Initialize the resource
   Store the resource number at the location provided by caller
Return true if new resource was created; else return false
```

**伪代码的好处**：
- 使审查更容易（不需看源代码）
- 支持迭代提炼
- 使更改更容易（几行伪代码比一页代码易改）
- 最小化注释工作（伪代码变成注释）
- 比其他形式设计文档更容易维护

## 使用PPP构建例程

### 设计例程
1. 检查前置条件
2. 定义例程将解决的问题
3. 命名例程
4. 决定如何测试
5. 研究标准库功能
6. 思考错误处理
7. 思考效率
8. 研究算法和数据类型
9. 编写伪代码
10. 检查伪代码

### 编码例程
1. 编写例程声明
2. 将伪代码转换为高层注释
3. 在每个注释下填写代码
4. 检查是否需要进一步分解

### 检查代码
1. Mental检查（桌面检查）
2. 编译
3. 在调试器中单步执行
4. 测试
5. 从例程中移除错误

### 清理收尾
- 检查接口
- 检查设计质量
- 检查变量
- 检查语句和逻辑
- 检查布局
- 检查文档
- 移除冗余注释

## PPP替代方案

| 方法 | 描述 |
|------|------|
| 测试先行开发 | 先编写测试用例再写代码 |
| 重构 | 通过语义保持转换改进代码 |
| 契约式设计 | 前置条件和后置条件 |
| 黑客方式 | 不推荐 |

## PPP检查清单

- [ ] 检查前置条件已满足
- [ ] 定义类将解决的问题
- [ ] 高层设计足够清晰
- [ ] 考虑如何测试
- [ ] 检查标准库和代码库
- [ ] 查阅算法参考书
- [ ] 用详细伪代码设计每个例程
- [ ] mental检查伪代码
- [ ] 准确翻译伪代码为代码
- [ ] 递归应用PPP
- [ ] 选择最好的而非第一次迭代

## 关键要点

1. 构建类和例程是迭代过程
2. 好的伪代码用英语、避免语言特征、在意图层次
3. PPP使编码容易，伪代码变注释确保注释准确
4. 不要满足于第一个设计
5. 在每一步检查工作
