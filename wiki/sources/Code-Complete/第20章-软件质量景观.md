## 摘要

本章概述软件质量技术及其在软件构建中的作用。核心观点：质量是免费的——提高质量降低成本；没有单一技术能完全有效；质量保证应贯穿整个开发过程。

## 软件质量特征

### 外部特征（用户关心）

| 特征 | 说明 |
|------|------|
| 正确性 | 精确按规格说明运行 |
| 可用性 | 易于学习和使用 |
| 效率 | 最小化资源使用 |
| 可靠性 | 长时间无故障运行 |
| 完整性 | 防止未授权访问 |
| 适应性 | 可用于不同环境 |
| 准确性 | 输出的精确程度 |
| 健壮性 | 异常条件下继续运行 |

### 内部特征（程序员关心）

- 可维护性、可变性
- 可移植性
- 可重用性
- 可读性
- 可测试性
- 可理解性

## 质量保证技术

**技术对比**：

| 技术 | 缺陷检测率 |
|------|-----------|
| 非正式设计评审 | 25-50% |
| 正式设计检查 | 45-70% |
| 非正式代码评审 | 20-35% |
| 正式代码检查 | 45-80% |
| 单元测试 | 15-35% |
| 集成测试 | 25-55% |
| 系统测试 | 25-65% |

**关键发现**：
- 无单一技术超过75%检测率
- 平均检测率约40%
- 技术组合比单一技术有效
- 检查比测试便宜且有效

## 修复成本

- 缺陷越早发现，修复成本越低
- 检查一步到位，测试需两步诊断
- IBM：检查修复3.5小时，测试修复12-25小时

## 质量与成本关系

**核心原则**：提高质量会降低开发成本

**原因**：
- 调试占用项目50%时间
- 减少调试时间是最有效的改进方式
- NASA：提高质量未增加总体成本
- IBM：缺陷最少项目进度最短

## 关键要点

1. 质量是免费的——提高质量降低成本
