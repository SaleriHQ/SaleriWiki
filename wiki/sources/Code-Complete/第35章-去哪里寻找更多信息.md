## 摘要

本章为软件开发提供了丰富的资源指引，包括软件构建书籍、软件工程概述、期刊杂志、以及软件开发者阅读计划。这是《Code Complete》的最后一章，将所有知识包装成一个完整的专业发展路径。

## 关键概念

### 软件构建核心书籍

| 书籍 | 作者 | 特点 |
|------|------|------|
| 实用程序员 | Hunt & Thomas | 专注于编码相关活动：测试、调试、断言 |
| 编程珍珠 | Bentley | 小型软件设计和编程的艺术与科学 |
| 极限编程解析 | Beck | 以构建为中心的开发方法 |
| 编写可靠代码 | Maguire | Microsoft Office 的 C 语言构建实践 |
| 编程实践 | Kernighan & Pike | 编程风格、设计、调试、测试 |

### 软件工程概述书籍

| 书籍 | 作者 | 特点 |
|------|------|------|
| 软件工程的事实与谬误 | Glass | 传统智慧的可读介绍 |
| 专业软件开发 | McConnell | 领域全景调查 |
| Swebok | Abran | 软件工程知识体系详细分解 |
| 计算机编程心理学 | Weinberg | 编程人因学 |
| 人月神话 | Brooks | IBM OS/360 项目案例研究 |
| 人件 | DeMarco & Lister | 团队人文因素 |

### 推荐期刊

**通俗期刊**（本地报刊亭有售）：
- 软件开发杂志 (Software Development)
- Dr. Dobb's Journal

**高雅期刊**（需订阅/图书馆）：
- IEEE Software - 最适合认真软件从业者
- IEEE Computer - 广泛主题，高质量
- ACM 通讯 - 最古老最受尊重

### 软件开发者阅读计划

**入门级别**（5本）：
- Conceptual Blockbusting - 概念性破冰
- Programming Pearls - 编程珍珠
- Facts and Fallacies of Software Engineering
- Software Project Survival Guide
- Code Complete

**从业者级别**（+8本）：
- Software Configuration Management Patterns
- UML Distilled
- Software Creativity
- Testing Computer Software
- Applying UML and Patterns
- Rapid Development
- Software Requirements
- NASA Manager's Handbook for Software Development

**专业级别**（+6本）：
- Software Architecture in Practice
- Refactoring
- Design Patterns
- Principles of Software Engineering Management
- Writing Solid Code
- Object-Oriented Software Construction

### 专业组织

| 组织 | 出版物 | 特点 |
|------|--------|------|

## 关键要点

1. **核心资源**：Code Complete + Programming Pearls + Facts and Fallacies 构成入门三件套
2. **持续学习**：软件信息快速变化，需要持续阅读跟上
3. **专业发展**：按级别递进阅读，从入门到专业
4. **实践结合**：阅读需与实践结合才能真正掌握
5. **社区参与**：加入专业组织与同行交流促进成长
