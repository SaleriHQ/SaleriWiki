## 摘要

本章讨论不常见的控制结构：多重返回、递归和 goto。核心观点：谨慎使用这些结构；递归适合小部分问题；goto 应作为最后手段。

## 多重返回

**使用原则**：
- 在能增强可读性时使用
- 使用守卫子句简化错误处理
- 最小化每个子程序的 return 数量

**守卫子句示例**：
```vb
' 坏：深度嵌套
If file.validName() Then
   If file.Open() Then
      If encryptionKey.valid() Then
         If file.Decrypt() Then
            ' 大量代码
         End If
      End If
   End If
End If

' 好：守卫子句
If Not file.validName() Then Exit Sub
If Not file.Open() Then Exit Sub
If Not encryptionKey.valid() Then Exit Sub
If Not file.Decrypt() Then Exit Sub
' 大量代码
```

## 递归

**适用场景**：
- 问题可分解为更小的相同问题
- 问题规模小
- 简单迭代难以实现

**递归原则**：
- 确保递归停止（有终止条件）
- 考虑使用安全计数器
- 递归限制在一个子程序中
- 关注栈使用
- 避免用递归计算阶乘/斐波那契

**示例**：
```cpp
// 迷宫路径搜索
bool FindPathThroughMaze(Maze maze, Point position) {
    if (AlreadyTried(maze, position)) return false;
    if (ThisIsTheExit(maze, position)) return true;
    RememberPosition(maze, position);
    
    if (MoveLeft(maze, position, &newPosition))
        if (FindPathThroughMaze(maze, newPosition)) return true;
    // ... 其他方向
    return false;
}
```

**安全计数器**：
```cpp
void RecursiveProc(int& safetyCounter) {
    if (safetyCounter > SAFETY_LIMIT) return;
    safetyCounter++;
    // ...
    RecursiveProc(safetyCounter);
}
```

## goto

### 反对 goto 的论点

- 代码质量与 goto 数量成反比
- goto 难以格式化
- 阻止编译器优化
- 违反代码从上到下流动原则

### 支持 goto 的论点

- 消除重复代码
- 资源分配/清理集中处理
- 某些情况下产生更小更快的代码
- 好的程序员会谨慎使用

### 错误处理和 goto

**带 goto 的清理模式**：
```vb
Sub PurgeFiles(ByRef errorState As Error_Code)
    MakePurgeFileList(fileList, numFilesToPurge)
    
    fileIndex = 0
    While fileIndex < numFilesToPurge
        fileIndex = fileIndex + 1
        If Not FindFile(fileList(fileIndex), fileToPurge) Then
            errorState = FileStatus_FileFindError
            GoTo END_PROC
        End If
        If Not OpenFile(fileToPurge) Then GoTo END_PROC
        If Not OverwriteFile(fileToPurge) Then GoTo END_PROC
        If Not Erase(fileToPurge) Then GoTo END_PROC
    Wend
    
END_PROC:
    DeletePurgeFileList(fileList, numFilesToPurge)
End Sub
```

### goto 替代方案

| 方法 | 优势 | 劣势 |
|------|------|------|
| 嵌套 if | 无 goto | 深度嵌套 |
| 状态变量 | 无 goto，无嵌套 | 额外测试 |
| try-finally | 结构化清理 | 需异常支持 |

## 关键要点

1. 多重返回可增强可读性，用于守卫子句
2. 递归适合可分解为相似子问题的情况
3. goto 应作为最后手段
4. 任何技术都应一致应用
