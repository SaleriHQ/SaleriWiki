## 摘要

表驱动方法是用查找表替代复杂逻辑的技术。核心观点：把程序知识放入数据而非逻辑；表提供逻辑和继承结构的替代方案；选择正确的访问方式。

## 为什么使用表

**优势**：
- 比复杂逻辑更简单
- 更容易修改
- 更高效
- 可在运行时从文件读取

**示例对比**：
```java
// 复杂逻辑
if ((('a' <= inputChar) && (inputChar <= 'z')) ||
    (('A' <= inputChar) && (inputChar <= 'Z'))) {
    charType = CharacterType.Letter;
} else if (...) { /* 更多条件 */ }

// 表驱动
charType = charTypeTable[inputChar];
```

## 访问方式

### 直接访问

适用于数据可直接作为索引的情况。

**每月天数示例**：
```vb
Dim daysPerMonth() = {31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31}
days = daysPerMonth(month - 1)
```

**多维费率表**：
```vb
Dim rateTable(SmokingStatus_Last, Gender_Last, MaritalStatus_Last, MAX_AGE)
rate = rateTable(smokingStatus, gender, maritalStatus, age)
```

### 索引访问

当主表太大或需要多维度访问时使用。

**结构**：
```
数据 → 索引表 → 主表
```

**优势**：
- 节省空间
- 支持多维度访问
- 易于维护

### 阶梯访问

适用于数据范围而非离散点。

**评分示例**：
```vb
Dim rangeLimit() = {50.0, 65.0, 75.0, 90.0, 100.0}
Dim grade() = {"F", "D", "C", "B", "A"}

gradeLevel = 0
While (gradeLevel < maxGradeLevel)
    If (studentScore < rangeLimit(gradeLevel)) Then
        studentGrade = grade(gradeLevel)
    End If
    gradeLevel = gradeLevel + 1
Wend
```

## 伪造查找键

当数据不能直接作为键时：

1. **复制信息**：为每个可能值设置条目
2. **转换键**：用 min/max 函数限制范围
3. **隔离转换**：放在单独函数中

```cpp
// 转换年龄为有效索引
int ageKey = max(min(66, age), 17);
// 使用 ageKey 直接访问表
```

## 表驱动消息处理

**概念**：用表描述消息格式，而非硬编码处理逻辑

```cpp
// 定义字段类型
enum FieldType { FloatingPoint, Integer, String, TimeOfDay, Boolean, BitField };

// 创建对象表
AbstractField* field[Field_Last];
field[FloatingPoint] = new FloatingPointField();
field[Integer] = new IntegerField();
// ...

// 通用消息处理
while (fieldIdx <= numFields) {
    fieldType = fieldDescription[fieldIdx].FieldType;
    field[fieldType].ReadAndPrint(fieldName, fileStatus);
}
```

**优势**：一个例程处理所有消息类型，无需为每种消息写独立代码

## 关键要点

1. 表是复杂逻辑和继承的替代方案
2. 选择访问方式：直接、索引或阶梯
3. 把知识放入数据而非逻辑
4. 表更灵活、更易维护
