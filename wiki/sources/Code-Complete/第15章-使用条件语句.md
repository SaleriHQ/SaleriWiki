## 摘要

本章讨论条件语句（if、else、case/switch）的最佳实践。核心观点：正常情况应放在 if 之后而非 else 之后；复杂测试应封装为布尔函数；case 语句应覆盖所有情况包括 default。

## if 语句

### 普通 if-then 语句

**编写原则**：
- 首先编写正常路径，异常情况放在后面
- 确保在相等性上正确分支
- 正常情况放在 if 之后，else 处理异常
- 空 if 子句改写为否定条件
- 考虑是否需要 else 子句

**示例对比**：
```vb
' 坏：正常与错误混在一起
If status = Error Then
   errorType = ...
Else
   ReadFile()  ' 正常
   If status = Success Then
      Summarize()  ' 正常
      If status = Error Then
         errorType = ...  ' 错误
      Else
         Print()  ' 正常
      End If
   End If
End If

' 好：正常在前，错误在后
If status = Success Then
   ReadFile()  ' 正常
   If status = Success Then
      Summarize()  ' 正常
      If status = Success Then
         Print()  ' 正常
      Else
         errorType = ...  ' 错误
      End If
   Else
      errorType = ...  ' 错误
   End If
Else
   errorType = ...  ' 错误
End If
```

## if-then-else-if 链

**使用建议**：
- 用布尔函数简化复杂测试
- 最常见情况放前面
- 确保覆盖所有情况
- 用 case 语句替代（如果语言支持）

**示例**：
```cpp
// 好：使用布尔函数
if ( IsLetter(inputChar) ) {
   characterType = CharacterType_Letter;
}
else if ( IsPunctuation(inputChar) ) {
   characterType = CharacterType_Punctuation;
}
```

## case 语句

**排序选择**：
| 排序方式 | 适用场景 |
|----------|----------|
| 字母/数字顺序 | 各 case 同等重要 |
| 正常情况在前 | 有主次之分 |
| 按频率排序 | 性能敏感 |

**使用建议**：
- 保持每个 case 动作简单
- 不编造虚假变量使用 case
- 合法使用 default 子句
- 在 C++/Java 中每个 case 后加 break
- 避免 fall-through（除非文档化）

**C++ Fall-through 文档化**：
```cpp
switch (errorLevel) {
   case DocumentationLevel_Full:
      DisplayErrorDetails();
      // FALLTHROUGH - 完整文档也打印摘要
   case DocumentationLevel_Summary:
      DisplayErrorSummary();
      // FALLTHROUGH - 摘要也打印错误号
   case DocumentationLevel_NumberOnly:
      DisplayErrorNumber();
      break;
}
```

## 检查表

- [ ] 代码正常路径清晰
- [ ] if-then 测试正确分支
- [ ] else 子句存在且有文档
- [ ] 正常情况跟随 if 而非 else
- [ ] 复杂测试封装为函数
- [ ] case 语句覆盖所有情况
- [ ] C/C++/Java 中每个 case 有 break

## 关键要点

1. 正常情况放 if 之后，异常情况放 else 之后
2. 用布尔函数替代复杂测试
3. 优先使用 case 而非 if-then-else-if 链
4. case 语句的 default 用于检测意外情况
5. C++/Java 中避免 fall-through
