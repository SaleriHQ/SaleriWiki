## 摘要

本章讨论直线代码（不含分支的代码）的组织原则。核心观点：语句应按依赖关系排列；相关语句应放在一起；使用块结构提高可读性。

## 语句排列原则

**按依赖关系排列**：
- 依赖语句在前
- 被依赖语句在后
- 保持依赖关系清晰

**示例**：
```cpp
// 好：依赖关系清晰
InitializeScore();
LoadHighScore();
DisplayScore();
UpdateStatistics();

// 坏：依赖关系混乱
DisplayScore();      // 依赖未初始化的数据
InitializeScore();
```

## 相关语句分组

**分组原则**：
- 相关操作放在一起
- 不相关操作分开
- 按功能/步骤组织

**组织方式**：
1. 按时间顺序（执行步骤）
2. 按功能分组
3. 按重要性排序
4. 按字母顺序（作为最后手段）

## 代码块结构

**块结构优点**：
- 视觉清晰
- 便于阅读
- 易于维护

**良好的缩进**：
```cpp
// 好：清晰的块结构
void ProcessOrder(Order& order) {
    ValidateOrder(order);
    CalculateTotal(order);
    ApplyDiscount(order);
    ProcessPayment(order);
    ShipOrder(order);
}
```

## 空格的使用

**水平空格**：
- 在赋值符周围加空格
- 在参数列表中加空格
- 缩进保持一致

**垂直空格**：
- 逻辑段落之间空行
- 相关语句紧密，组之间分开
- 不要过度使用

## 显式vs隐式

**显式优于隐式**：
```cpp
// 隐式：不清晰
if (x) DoSomething();

// 显式：清晰
if (x != 0) {
    DoSomething();
}
```

**原则**：
- 表达式写完整
- 循环条件明确
- 条件语句加括号

## 特定语句组织

### 简单语句

- 一行一条语句
- 不在多语句间隐藏赋值

### 复合语句

- if/for/while 的 body 总是加花括号
- 即使 body 只有一条语句

### 条件语句

- 先处理正常情况
- 后处理异常
- 条件表达式清晰

## 关键要点

1. 按依赖关系排列语句
2. 相关语句放在一起
3. 使用块结构提高可读性
4. 善用空格分隔逻辑段落
5. 显式优于隐式
6. 条件 body 始终使用花括号
