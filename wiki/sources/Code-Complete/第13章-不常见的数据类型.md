## 摘要

本章讨论不常见但重要的数据类型：结构体、指针和引用、全局数据。核心观点：结构体将相关数据组织在一起；指针是强大但危险的工具；全局数据应谨慎使用。

## 结构体

**定义**：将相关数据组合成单一类型的用户定义类型

**好处**：
- 组织相关数据
- 提高可读性
- 简化参数传递
- 表达真实世界实体

**示例**：
```cpp
struct Customer {
    string name;
    string address;
    string phone;
    double balance;
};
```

**设计原则**：
- 数据应属于一起
- 结构体大小适中
- 避免上帝结构体
- 封装操作与数据

## 指针和引用

### 指针基础

**指针操作**：
- 赋值
- 解引用
- 取地址
- 指针算术

**指针问题**：
- 未初始化指针
- 野指针（dangling pointers）
- 内存泄漏
- 缓冲区溢出

### 指针安全实践

**防御性指针操作**：
```cpp
// 检查指针有效性
if (ptr != nullptr) {
    // 安全使用
}

// 初始化所有指针
int* ptr = nullptr;

// 删除后置空
delete ptr;
ptr = nullptr;

// 使用智能指针（C++）
auto ptr = std::make_unique<int>();
```

### 引用（C++）

**引用特点**：
- 必须初始化
- 始终有效
- 不能重新赋值
- 无空值检查开销

**使用场景**：
```cpp
// 函数参数使用引用避免拷贝
void ProcessCustomer(const Customer& c);

// 运算符重载
class Vector {
    int& operator[](int index);
};
```

### 指针 vs 引用

| 特性 | 指针 | 引用 |
|------|------|------|
| 空值 | 可为 null | 必须有效 |
| 重新赋值 | 可重新赋值 | 不可重绑定 |
| 语法 | 用 * 解引用 | 直接使用 |
| 用途 | 可选参数 | 必须参数 |

## 全局数据

**全局数据问题**：
- 难以追踪修改
- 并发访问问题
- 难以测试
- 难以理解依赖

**全局数据使用情况**：
- 全局常量（通常可接受）
- 配置信息
- 性能关键数据
- 日志系统

### 减少全局数据

**替代方案**：
```cpp
// 替代1：封装到类中
class AppConfig {
public:
    static AppConfig& Instance() {
        static AppConfig instance;
        return instance;
    }
private:
    AppConfig() {}
};

// 替代2：依赖注入
void ProcessData(Config& config) {
    // 使用注入的配置
}
```

**如果必须使用全局数据**：
- 使用 getter/setter
- 文档化所有访问点
- 最小化数量
- 使用命名约定标识

## 内存位置

| 类型 | 特点 |
|------|------|
| 栈 | 自动管理，快速 |
| 堆 | 手动管理，灵活 |
| 静态/全局 | 整个生命周期存在 |

## 关键要点

1. 结构体组织相关数据，提高可读性
2. 指针是强大的工具但需要谨慎
3. 初始化所有指针，删除后置空
4. 优先使用引用而非指针
5. 全局数据应谨慎使用，优先封装
6. 理解内存位置（栈、堆、全局）
