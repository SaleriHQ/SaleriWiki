## 摘要

本章讨论基本数据类型的正确使用：数字、整数、浮点数、字符串和布尔值。核心观点：避免使用字面量，使用命名常量；小心浮点数的比较；字符串是特殊的数据类型。

## 数字

### 整数

**注意事项**：
- 检查整数溢出
- 注意除法精度丢失
- 考虑使用无符号整数时的陷阱

**溢出检测示例**：
```cpp
// 检测溢出
if (a > INT_MAX - b) {
    // 处理溢出
}
```

### 浮点数

**精度问题**：
```cpp
// 坏：浮点数比较
if (a == b)  // 可能永远不等

// 好：使用容差
if (abs(a - b) < 0.0001)
```

**浮点数陷阱**：
- 不要用浮点数表示货币
- 累积误差
- 不要在循环中比较相等

**建议**：
- 用整数表示货币（分、厘分）
- 使用高精度库处理货币
- 了解语言的浮点实现

## 字符串

**常见操作**：
- 初始化
- 比较
- 连接
- 搜索
- 转换
- 长度检查

**安全实践**：
- 使用安全字符串函数
- 避免缓冲区溢出
- 显式指定缓冲区大小
- 使用现代语言的安全特性

**C/C++ 字符串处理**：
```cpp
// 使用安全函数
strncpy(dest, src, sizeof(dest) - 1);
dest[sizeof(dest) - 1] = '\0';

// C++ 使用 string
std::string s = "hello";
```

## 布尔

**使用建议**：
- 使用布尔进行逻辑判断
- 将复杂判断封装为布尔函数
- 布尔变量使用 is、has、can、should 前缀

**布尔命名**：
```cpp
bool isReady;      // ✓ 好
bool found;        // ✓ 可接受
bool statusFlag;   // ✗ 坏
```

## 枚举

**优点**：
- 提高可读性（Color.Red vs 1）
- 支持编译器检查
- 易于扩展

**使用模式**：
```cpp
enum Color {
    Color_Red,
    Color_Green,
    Color_Blue
};

// 或使用类枚举
enum class Color {
    Red,
    Green,
    Blue
};
```

## 命名常量

**避免字面量**：
```cpp
// 坏
if (index < 100) { ... }

// 好
const int MAX_ITEMS = 100;
if (index < MAX_ITEMS) { ... }
```

**常量命名约定**：
- 全大写 + 下划线：MAX_ITEMS
- 或使用 const：const int MaxItems = 100

## 数组

**边界检查**：
- 验证索引在有效范围内
- 优先使用安全容器
- 注意 off-by-one 错误

## 关键要点

1. 避免字面量，使用命名常量
2. 浮点数比较使用容差
3. 货币计算用整数（分、厘分）
4. 字符串操作注意缓冲区安全
5. 布尔变量使用肯定命名
6. 枚举提高可读性和安全性
