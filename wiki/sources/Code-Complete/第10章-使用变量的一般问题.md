## 摘要

本章讨论变量使用的基本原则：隐式声明、变量初始化、作用域和持久性。核心观点：变量初始化是编程中最常见的错误来源之一；作用域最小化使变量更易追踪和理解；按距离原则，密切相关变量应放在一起。

## 隐式声明

**问题**：隐式声明变量会导致：
- 拼写错误变成新变量
- 意外创建全局变量
- 代码可读性差

**建议**：
- 启用编译器警告
- 声明所有变量
- 遵循语言约定

## 变量初始化

**初始化错误的原因**：
- 变量从未赋值
- 变量被赋予错误值
- 旧值影响新计算
- 超出范围的索引
- 忘记初始化例程

**初始化原则**：
- 在声明时初始化
- 使用final/const防止意外修改
- 检查数组边界
- 在构造函数中初始化成员变量

## 作用域

**作用域原则**：使变量对尽可能少的代码可见

**作用域技术**：
- 减小作用域直到"刚好够用"
- 按距离原则：密切相关变量放在一起
- 避免全局变量
- 减少生命周期长度

**作用域阶段**：
| 阶段 | 说明 |
|------|------|
| 1 | 所有变量全局 |
| 2 | 按字母顺序分组 |
| 3 | 按功能分组 |
| 4 | 相关变量就近声明 |
| 5 | 例程内变量 | 
| 6 | 复合语句内变量 |

## 持续时间（生命周期）

**概念**：变量存在的时间长度

**注意事项**：
- 避免保留对象（stale objects）
- 小心线程安全
- 最小化全局变量持续时间
- 对象生命周期要明确

## 绑定时间

**定义**：变量与其值绑定的时刻

**类型**：
| 时间点 | 说明 | 风险 |
|--------|------|------|
| 代码编写时 | 硬编码常量 | 缺乏灵活性 |
| 运行时绑定 | 从配置/输入读取 | 更灵活 |
| 对象创建时 | 构造函数中设置 | 中等 |
| 第一次使用时 | 延迟初始化 | 复杂 |

**建议**：使用命名常量而非 literals
