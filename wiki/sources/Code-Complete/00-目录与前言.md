## 摘要

本文档是《Code Complete》第二版的目录和前言部分，由 Steve McConnell 撰写。前言阐述了写作背景：构建主题一直被忽视、构建很重要、没有可比的书籍。目录展示了全书的7个部分共35章的结构。

## 书籍结构

| 部分 | 主题 | 章节数 |
|------|------|--------|
| 第一部分：奠定基础 | 隐喻、准备、构建决策 | 4章 |
| 第二部分：创建高质量代码 | 设计、类、例程、防御式编程、伪代码 | 5章 |
| 第三部分：变量 | 数据类型、命名、作用域 | 4章 |
| 第四部分：语句 | 控制结构、表驱动方法 | 6章 |
| 第五部分：代码改进 | 质量、协作、测试、调试、重构、调优 | 7章 |
| 第六部分：系统考虑 | 规模、管理、集成、工具 | 4章 |
| 第七部分：软件工艺 | 布局、文档、性格、主题、资源 | 5章 |

## 前言核心观点

**为什么写这本书**：
- 研究到实践的差距：5-15年的滞后
- 构建主题一直被忽视
- 没有可比的书籍
- 最佳实践与一般实践差距巨大

**构建的重要性**：
- 小型项目：构建占65%工作量、75%错误
- 中型项目：构建占50%工作量、50-75%错误
- 构建是保证会完成的唯一活动

**本书特点**：
- 完整的软件构建参考
- 可直接使用的检查清单
- 最新技术信息
- 大量代码示例（近500个）
- 平衡讨论优缺点，无炒作

## 关键要点

1. 构建是软件生命周期中最熟悉的活动
2. 研究发现的最佳实践需要5-15年才能进入商业实践
3. 代码构建是改进开发实践的独特丰饶领域
4. 本书填补了入门和高级编程文本之间的空白
