## 摘要

本章讨论调试的科学方法和技巧。核心观点：调试能力差异可达20:1；使用科学方法系统调试；修复问题而非症状。

## 调试性能差异

| 指标 | 最快3人 | 最慢3人 |
|------|--------|--------|
| 平均调试时间 | 5分钟 | 14分钟 |
| 未发现缺陷 | 0.7 | 1.7 |
| 引入新缺陷 | 3.0 | 7.7 |

## 科学调试方法

**步骤**：
1. 稳定错误：让缺陷可靠重现
2. 定位错误源：
   - 收集产生缺陷的数据
   - 分析数据形成假设
   - 设计实验证明/反驳假设
   - 执行实验验证
3. 修复缺陷
4. 测试修复
5. 寻找类似错误

## 寻找缺陷的技巧

**收集数据**：
- 使用所有可用数据形成假设
- 完善测试用例
- 用不同方式重现错误
- 生成更多数据产生更多假设

**缩小范围**：
- 分而治之搜索
- 增量集成
- 检查最近修改的代码
- 检查常见缺陷

**心理技巧**：
- 和他人讨论（忏悔式调试）
- 暂时放下问题
- 头脑风暴可能假设

## 蛮力调试

当其他方法失败时使用：
- 完整代码审查
- 重写代码部分
- 用最高警告级别编译
- 在调试器中单步执行
- 用不同编译器测试

## 修复缺陷的原则

**原则**：
- 修复前理解问题
- 理解程序而不仅是问题
- 确认诊断正确
- 放松，不要急
- 保存原始代码
- 修复问题而非症状
- 一次只做一个更改
- 检查修复的副作用
- 添加暴露缺陷的测试
- 寻找类似错误

**修复症状vs修复问题**：
```java
// 坏：修复症状
if (client == 45) {
    sum[45] = sum[45] + 3.45;  // 明天可能出错
}

// 好：修复根本问题
```

## 关键要点

1. 好与差的调试性能差异10:1以上
2. 使用科学方法系统调试
3. 修复前理解根本问题
4. 修复问题而非症状
5. 编译器警告设为最高级别
