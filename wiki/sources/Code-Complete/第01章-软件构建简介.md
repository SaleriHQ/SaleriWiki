## 摘要

本章是《Code Complete》的开篇，定义了什么是软件构建，并阐明了构建在软件开发中的核心地位。构建不仅仅是编码和调试，还包括详细设计、单元测试、集成测试等活动。研究表明构建占用项目30%-80%的时间，是保证会完成的唯一活动。

## 什么是软件构建

**构建活动包括**：
- 验证基础工作已就绪
- 确定如何测试代码
- 设计和编写类和例程
- 创建和命名变量及常量
- 选择控制结构并组织语句块
- 单元测试、集成测试和调试
- 代码审查
- 格式化、注释和代码完善
- 软件组件集成
- 代码调优（性能优化）

**非构建活动**：管理、需求开发、架构设计、UI设计、系统测试、维护

## 为什么构建很重要

| 原因 | 说明 |
|------|------|
| 时间占比高 | 占项目总时间30%-80% |
| 处于中心位置 | 需求和架构在上游，系统测试在下游 |
| 生产力差异大 | 个人程序员生产力差异可达10-20倍 |
| 代码是唯一准确描述 | 源代码通常是最新的唯一文档 |
| 保证完成 | 构建是唯一保证会完成的活动 |

## 关键要点

1. 构建是软件开发的中心活动，是每个项目保证会发生的唯一活动
2. 构建中的主要活动是详细设计、编码、调试、集成和开发人员测试
3. 构建的质量实质性地影响软件的质量
4. 如何进行构建的理解决定了你作为程序员有多好
