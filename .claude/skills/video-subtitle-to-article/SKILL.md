# Video Subtitle to Article

## 🎯 任务目标
将 `Clippings/VideoVTT/` 目录下的字幕文件转换为：
👉 **高质量总结 + 可读性强 + 结构清晰的 Markdown 技术文章**

⚠️ 注意：不是转录字幕，而是**理解 → 重构 → 输出**

完成之后将字幕文件归档到`Clippings/VideoVTT/归档/`

---

## 📂 输入格式

字幕文件格式：

```

0:00
Today we're going to take a look at...

0:10
Next we will...

```

特征：
- 时间戳 + 文本
- 段落分隔

---

## 🧠 处理流程（核心逻辑）

### 1️⃣ 读取文件
读取：
```

Clippings/VideoVTT/{filename}.md

````

---

### 2️⃣ 内容理解（非常关键）

你必须：

- 理解视频主题
- 识别结构（引入 / 核心讲解 / demo / 总结）
- 提取：
  - 技术点
  - 概念
  - 操作步骤
  - 代码（如有）

---

### 3️⃣ 内容重构原则（⚠️ 强约束）

❌ 禁止：
- 逐句翻译字幕
- 保留口语废话（um, you know, etc）

✅ 必须：
- 使用**技术文章风格**
- 信息压缩 + 逻辑增强
- 合理补全上下文

---

## 🏷️ 标题生成规则

优先级：

1. 从文件名提取
2. 基于内容优化（推荐）

示例：

```

NeoVim 0.12 Finally Gives You a Real Plugin Manager
→
Neovim 0.12 内置插件管理器深度解析

```

---

## 🏷️ 标签生成规则

至少包含：

- 技术标签（如 neovim / vim / cli）
- 内容标签（tutorial / guide）
- 通用标签：
  - video
  - tutorial

---

## 💾 输出要求

- 保存到：
```

Clippings/{slug}.md

````

- slug 规则：
  - 小写英文
  - 用 `-` 分隔
  - 示例：
    ```
    neovim-plugin-manager-guide.md
    ```

---

## 🚫 常见错误（必须避免）

❌ 字幕直译  
❌ 无结构大段文本  
❌ 缺少核心总结  
❌ 没有提炼知识点  

---

## ✅ 输出质量标准（重要）

生成内容应：

- 像技术博客，而不是字幕整理
- 适合二次阅读（知识沉淀）
- 逻辑清晰（章节分明）
- 信息密度高

---

## 💡 进阶优化（可选）

如果内容复杂，可以：

- 增加章节（如 原理解析 / 架构设计）
- 添加对比（旧方案 vs 新方案）
- 总结“适用场景”
