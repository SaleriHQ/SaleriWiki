## 多轮面试管理系统

---

## 1. 引言

### 1.1 目的

本文档旨在详细定义多轮面试管理系统的功能需求、非功能需求及技术约束，作为开发团队、测试团队和项目干系人之间的共识基准，确保最终交付的系统满足企业面试管理的全部业务需求。

### 1.2 范围

本文档涵盖以下范围：
- 候选人从简历投递到最终录用/拒绝的全生命周期管理
- 四轮面试（初试、复试、HR面、终面）的流程管理与数据记录
- 多角色权限控制体系
- 面试评价的完整记录与汇总展示
- 系统的基础架构与技术选型要求

## 2. 总体描述

### 2.1 产品 Perspective

多轮面试管理系统是一款面向企业人力资源部门的Web应用，用于管理候选人的面试全流程。系统模拟真实面试场景，支持面试官在不同阶段记录候选人的表现、评分和评价，并提供完整的数据汇总和决策支持。

系统采用前后端分离架构，前端基于Vue3生态，后端使用JSON Server模拟RESTful API，支持快速原型开发和迭代验证。

### 2.2 产品功能概述

| 功能模块 | 核心功能 |
|----------|----------|
| 认证模块 | 登录、退出、Token管理、路由守卫 |
| 候选人管理 | 创建、编辑、删除、列表展示、筛选、详情查看 |
| 初试管理 | 技术基础问答记录、逐题打分、基础知识评估 |
| 复试管理 | 技术深度问答、深问追问记录、编程题评价、能力雷达图 |
| HR面管理 | 薪资谈判记录、稳定性评估、软技能评分、风险标记 |
| 终面管理 | 前面轮次汇总、高层决策、Offer审批 |
| 评价管理 | 各轮评价查看、汇总对比、时间线展示 |
| 用户权限 | 角色管理、权限配置、动态菜单 |
| 面试官管理 | 面试官列表、角色分配 |

### 2.3 用户特征

| 角色 | 描述 | 权限范围 |
|------|------|----------|
| 管理员 (Admin) | 系统最高权限，可管理所有功能 | 全部功能 |
| 面试官 (Interviewer) | 负责技术面试，仅能面试指定轮次 | 初试、复试，仅查看本人评价 |
| HR专员 (HR) | 负责候选人信息管理、HR面 | 候选人管理、HR面、查看全部评价 |
| 高层管理 (Manager) | 最终决策者 | 终面、查看全部、审批Offer |

### 2.4 约束

#### 4.1 设计约束
- 技术栈固定：Vue3 + Vite + TypeScript + Pinia + Vue Router
- 数据存储：JSON Server Mock（可切换为真实后端）
- 浏览器支持：Chrome 90+、Firefox 88+、Safari 14+、Edge 90+
- 响应式要求：优先桌面端，兼顾平板

#### 4.2 操作约束
- 单一候选人同时只能进行一轮面试
- 候选人状态为"淘汰"或"录用"后不可变更
- 面试评价提交后不可修改（仅管理员可撤销）
- 评分数据需在面试结束后24小时内提交

### 2.5 假设与依赖

1. 假设用户具备基本的计算机操作能力
2. 假设系统运行在企业内部网络环境
3. 依赖 JSON Server 提供稳定的 Mock API
4. 依赖 Vue3 生态库的稳定性和兼容性
5. 假设面试流程为企业标准的四轮面试

## 3. 功能需求

### 3.1 候选人管理

#### 3.1.1 创建候选人
- 支持手动创建候选人信息
- 必填字段：姓名、手机号、应聘岗位
- 选填字段：邮箱、简历附件、教育经历、工作经历
- 创建后自动生成唯一候选人ID
- 创建后状态默认为"待面试"

#### 3.1.2 候选人列表
- 表格展示候选人列表
- 支持分页（每页20条）
- 支持按姓名、手机号、岗位、状态、面试官筛选
- 支持按创建时间排序
- 每行显示：姓名、岗位、当前轮次、状态、面试官、下轮时间、操作

#### 3.1.3 候选人详情
- Tab页展示：基本信息、面试记录、评价汇总、Timeline
- 基本信息：个人信息、简历附件
- 面试记录：各轮面试卡片列表
- 评价汇总：各轮评分雷达图对比
- Timeline：完整面试进度可视化

#### 3.1.4 状态流转
```
新建 → 待面试 → 面试中 → [初试通过/淘汰] → [复试通过/淘汰] → [HR通过/淘汰] → [录用/拒绝]
```

### 3.2 初试管理

#### 3.2.1 基础问答
- 系统预置题目库（可扩展）
- 题目分类：HTML/CSS、JavaScript、框架基础、网络、工程化
- 每题记录：题目、期望答案、候选人回答、得分、备注
- 得分范围：0-3分（0=不会，1=错误，2=基本正确，3=完美）

#### 3.2.2 评分维度
| 维度 | 分值范围 | 说明 |
|------|----------|------|
| HTML/CSS | 0-10 | 语义化、布局、盒模型 |
| JavaScript | 0-15 | 原型链、异步、作用域 |
| 框架基础 | 0-10 | Vue/React核心概念 |
| 网络基础 | 0-10 | HTTP、跨域、缓存 |
| 工程化 | 0-5 | Webpack、Git、CI/CD |

#### 3.2.3 决策操作
- 保存草稿：仅保存，不更新状态
- 淘汰：候选人状态变为"已淘汰"
- 通过并进入下一轮：候选人进入复试阶段

### 3.3 复试管理

#### 3.3.1 技术深度问答
- 话题分类：架构设计、性能优化、原理源码、项目经验、故障排查
- 每题支持追问链（最多3层）
- 评分维度：深度(0-5)、表达清晰度(0-5)、逻辑性(0-5)

#### 3.3.2 编程题评价
- 记录：题目名称、难度、候选人代码、时间花费、运行结果
- 结果选项：AC(通过)、WA(错误)、TLE(超时)、未完成

#### 3.3.3 能力雷达图
- 五维能力评估：前端性能、工程架构、问题排查、代码质量、技术深度
- 每维度0-5分
- 可视化雷达图展示

### 3.4 HR面管理

#### 3.4.1 薪资与稳定性
- 当前薪资（数字输入）
- 期望薪资（数字输入）
- 薪资是否可协商（开关）
- 在职时长、下家离职原因、过往跳槽次数
- 稳定性评分(1-5)

#### 3.4.2 软技能评估
- 沟通表达能力(1-5)
- 价值观匹配度(1-5)
- 职业发展规划（文本）
- 综合推荐度：强烈推荐/推荐/存疑/不推荐

#### 3.4.3 风险标记
- 频繁跳槽（2年内超过2次）
- 空窗期超过6个月
- 薪资期望明显超出范围
- 态度消极

### 3.5 终面管理

#### 3.5.1 前面轮次汇总
- 展示初试、复试、HR面的评分和结论
- 亮点与不足的提炼
- 薪资期望对比

#### 3.5.2 高层决策
- 综合评分(1-5)
- 核心优势勾选
- 顾虑点勾选
- 决策选项：录用/待定/拒绝/薪资谈判后再决定

#### 3.5.3 Offer审批
- 拟Offer薪资
- 职级（P5/P6/P7）
- 预计入职日期
- 批复意见
- 批复人签名

### 3.6 权限管理

#### 3.6.1 角色管理
- 系统预置角色：Admin、面试官、HR、高层
- 支持查看角色列表
- 支持分配权限

#### 3.6.2 权限配置
- 菜单权限：控制侧边栏显示
- 操作权限：控制按钮显隐
- 数据权限：控制能看到的数据范围

#### 3.6.3 动态菜单
- 根据用户角色动态生成侧边栏
- 根据权限过滤路由

## 5. 数据需求

### 5.1 主要实体

#### 5.1.1 候选人 (Candidate)
```
- id: string (UUID)
- name: string
- email: string
- phone: string
- position: string
- status: 'pending' | 'interviewing' | 'passed' | 'rejected' | 'hired'
- currentRound: 'initial' | 'second' | 'hr' | 'final' | null
- resumeUrl: string
- createdAt: string
- updatedAt: string
```

#### 5.1.2 面试记录 (InterviewRecord)
```
- id: string
- candidateId: string
- round: 'initial' | 'second' | 'hr' | 'final'
- interviewerId: string
- status: 'pending' | 'completed' | 'skipped'
- scheduledAt: string
- completedAt: string
```

#### 5.1.3 评价 (Evaluation)
```
- id: string
- recordId: string
- scores: object
- comment: string
- recommendation: 'strong_pass' | 'pass' | 'weak_pass' | 'reject'
- createdAt: string
```

#### 5.1.4 用户 (User)
```
- id: string
- username: string
- password: string (hashed)
- name: string
- role: 'admin' | 'interviewer' | 'hr' | 'manager'
- email: string
- createdAt: string
```

### 5.2 数据关系图

```
User (面试官) 1 ──N InterviewRecord
Candidate 1 ──N InterviewRecord
InterviewRecord 1 ──1 Evaluation
Candidate 1 ──N TimelineEvent
User 1 ──N Role ──N Permission
```

## 6. 验收标准

### 6.1 功能验收

| 功能 | 验收条件 |
|------|----------|
| 登录 | 输入正确账号密码跳转首页，错误提示"账号或密码错误" |
| 候选人创建 | 必填项为空时禁止提交，创建成功后跳转列表 |
| 面试评分 | 所有必填项填写后才能提交，提交后状态更新 |
| 状态流转 | 淘汰后不可再操作，通过后进入下一轮 |
| 权限控制 | 无权限用户看不到对应菜单和按钮 |
| 动态菜单 | 切换角色后侧边栏相应变化 |

### 6.2 界面验收

| 检查点 | 要求 |
|--------|------|
| 布局一致性 | 所有页面布局风格统一 |
| 响应式 | 1920px、1366px、768px下布局正常 |
| 交互反馈 | 按钮hover/active状态正确 |
| 加载状态 | 网络请求时显示loading |
| 空状态 | 无数据时显示友好提示 |
| 错误状态 | 表单验证错误清晰提示 |

### 6.3 性能验收

| 测试场景 | 达标条件 |
|----------|----------|
| 首页加载 | Lighthouse Performance ≥ 80 |
| 列表加载100条 | 渲染时间 ≤ 1s |
| 表单提交 | 响应时间 ≤ 1s |
| 雷达图渲染 | 渲染时间 ≤ 200ms |
