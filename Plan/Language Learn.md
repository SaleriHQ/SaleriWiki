# Language Learn - 项目规划

## 起因

市面上的软件收费不好用，开源的软件老旧、需要重构，集成性差。

## 目标

制作一个**私人的语言学习 SaaS 平台**，为用户提供服务，支持会员制。

---

# 一、项目概览

## 1. 项目定位

多租户 SaaS 语言学习平台，支持：

- 📚 PDF / EPUB 阅读
- 📖 单词/句子标注
- 📘 服务端词典查词（**初期支持英语，后续扩展多语言**）
- 🖼 图片辅助记忆
- 🧠 Anki 导出
- 👥 会员制服务
- 🔐 用户认证与数据隔离

### 语言支持策略

| 阶段 | 语言 | 说明 |
|------|------|------|
| **V1** | 英语 | 初期只支持英语词典 |
| **V2** | 日语、韩语等 | 扩展多语言词典支持 |

**设计原则**：词典服务采用插件化架构，后续新增语言只需部署对应词典文件。

## 2. 技术栈

| 层级 | 技术选型 |
|------|---------|
| 前端 | Nuxt 3 (SPA) + Vue 3 + SCSS + Pinia |
| 阅读引擎 | epub.js (EPUB) + pdf.js (PDF) |
| 后端 | **待定**：Go (Hono/Gin) **/** Rust (Axum/Actix) |
| 数据库 | PostgreSQL + Redis |
| ORM | SQLx (Go) / Diesel / sqlx (Rust) |
| 存储 | 服务器本地 + Cloudflare R2（可选） |
| 部署 | 自托管 VPS（阿里云/腾讯云/Vultr）+ Cloudflare |

### 后端语言选择

| 语言 | 框架 | 优点 | 缺点 | 推荐场景 |
|------|------|------|------|----------|
| **Go** | Hono / Gin / Echo | 上手快、生态成熟、部署简单 | 性能中等 | MVP 快速开发 |
| **Rust** | Axum / Actix-web | 性能极致、内存安全 | 学习曲线陡 | 性能优化阶段 |

> **选择建议**：先用 Go 快速验证产品，后续可根据需要用 Rust 重构性能敏感模块（词典查询）。

# 四、功能模块设计

## 1. 书籍管理模块

### 功能

- 上传 EPUB / PDF 文件到 S3
- 服务端解析提取纯文本
- 提取书籍元数据（标题、作者、封面）
- 统计总词数、不重复词数

### 数据表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 用户 ID |
| title | VARCHAR | 书名 |
| author | VARCHAR | 作者 |
| file_url | TEXT | S3 存储地址 |
| file_type | VARCHAR | epub / pdf |
| content | TEXT | 解析后的纯文本 |
| total_words | INT | 总词数 |
| unique_words | INT | 不重复词数 |
| cover_url | TEXT | 封面图片 |
| language | VARCHAR | 书籍语言 |
| reading_progress | FLOAT | 阅读进度 0-100% |

## 2. 阅读器模块

### 功能

- 加载 EPUB / PDF 书籍
- 文本选择（单词 / 句子）
- 高亮标注
- **智能单词标注**：根据用户词库自动区分颜色

### 阅读器特色

| 功能 | 说明 |
|------|------|
| 单词高亮 | 🟢 绿色=已学习 🔵 蓝色=待复习 🟡 黄色=生词 |
| 点击查词 | 点击任意单词弹出词典释义 |
| 进度同步 | 自动保存阅读位置，多设备同步 |

### 组件结构

```
components/
├── EpubReader.vue        # EPUB 阅读器
├── PdfViewer.vue         # PDF 阅读器
├── TextSelection.vue     # 文本选择
├── WordHighlight.vue     # 单词智能标注
└── DictionaryPopover.vue # 查词弹窗
```

## 3. 标注系统

### 功能

- 单词标注：点击查词 → 保存到个人词库
- 句子标注：高亮段落 + 翻译 + 笔记
- 书籍内标注：在阅读位置保存标注

### 数据表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 用户 ID |
| book_id | UUID | 书籍 ID |
| type | VARCHAR | word / sentence |
| content | TEXT | 标注内容 |
| context | TEXT | 上下文 |
| translation | TEXT | 翻译 |
| position | JSONB | 位置信息 |

## 4. 词典服务（服务端部署）

### 设计理念

- **服务端部署**：词典数据存储在服务器
- **在线查词**：用户查询时调用 API，无需第三方
- **隐私保护**：查询数据不泄露
- **多语言支持**：可部署多个词典

### 词典格式

| 格式 | 特点 | 生态 |
|------|------|------|
| MDict (.mdx) | 流行、压缩率高 | 词典资源丰富 |
| StarDict (.dict) | 老牌格式 | 工具较多 |

### 服务架构

```
前端 ──── 查询 "hello" ────► 后端词典服务 ────► 本地词典文件
      ◄── 返回释义、音标 ◄────────────────────
```

### 开源词典资源

| 词典 | 语言对 | 大小 | 来源 |
|------|--------|------|------|
| ECDICT | 英汉 | ~50MB | github.com/skywind3000/ECDICT |
| JLPT 词典 | 日汉 | ~30MB | MDict 社区 |

**初期建议**：只集成 ECDICT（免费开源）

## 5. 单词标注系统

### 功能

根据用户词库，在阅读时自动标注单词状态：

| 状态 | 颜色 | 条件 |
|------|------|------|
| 已学习 | 🟢 绿色 | 用户词库中已掌握 |
| 待复习 | 🔵 蓝色 | 用户词库中需要复习 |
| 待学习 | ⚪ 默认 | 单词不在词库中 |
| 生词 | 🟡 黄色 | 高频必学词汇（基于词表匹配） |

### 标注逻辑

1. 获取用户已学单词集合
2. 提取阅读文本中的所有单词
3. 过滤已学习的单词
4. 与常用词表比对（高频词表，如 CEFR A1-C2）
5. 标记需要学习的单词（基于规则，非 AI）

## 6. 单词管理系统

### 功能

- 单词本：查看所有保存的单词
- 标注列表：按书籍/日期筛选
- 复习状态：掌握程度、复习次数

### 数据表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 用户 ID |
| word | TEXT | 单词 |
| translation | TEXT | 翻译 |
| pronunciation | TEXT | 发音 |
| images | TEXT[] | 图片列表 |
| mastery_level | INT | 掌握程度 0-5 |
| review_count | INT | 复习次数 |
| next_review_at | TIMESTAMP | 下次复习时间 |

## 7. 会员系统

### 数据表

| 表名 | 说明 |
|------|------|
| users | 用户信息、会员等级、存储配额 |
| subscriptions | Stripe 订阅记录 |

## 8. 导出模块

### 功能

- Anki 导出：生成 CSV / APKG 文件
